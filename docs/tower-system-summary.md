# 타워 시스템 정리 (Tower Defense)

`Pipeline Defense Farm`의 타워 시스템을 현재 문서 기준으로 요약한 표입니다.

## 1) 타워 타입 요약

| 타워명 | 역할 | 강점 | 리스크/제약 | 추천 구간 |
|---|---|---|---|---|
| 효소 분사기 (Enzyme) | 특정 음식 타입 분해 | 탄수/단백/지방 등 타입 상성 대응 | 범용성은 조합 의존 | 전 구간 기본 코어 |
| 위산 분무 (Acid) | 소화 피해 + 산도 기반 처리 | 위(Stomach)에서 높은 효율 | 과산 누적 리스크 | 위 |
| 담즙 제트 (Bile) | 지방 유화/약화, 범위 디버프 | 지방식/고지방 웨이브 대응 | 단일 고속 처치보다는 보조 성격 | 위, 소장 |
| 프로바이오틱스 구름 (Probiotic) | 장내 환경 버프, 부패/발효 완화 | 트러블 완화에 강함 | 즉발 화력 낮음 | 대장 |
| 연동 운동 펄스 (Peristalsis) | 음식 밀어내기/정체 해소 | 라인 압력 관리, 막힘 완화 | 직접 딜 기여 낮음 | 정체 구간 전체 |
| 흡수 필터 (Absorption) | 흡수 보조 + 자원 획득 보정 | 전투+경제 하이브리드 | 성장형 성격(초반 효율 변동) | 소장 |

## 2) 타워 공통 시스템

| 항목 | 내용 |
|---|---|
| 설치 위치 | Buildable Tile(빌드 가능 타일)에 배치 |
| 기본 기능 | 음식 대상 소화 피해 또는 상태이상 적용 |
| 상태이상 축 | 분해 / 중화 / 유화 / 흡수 촉진 |
| 목표 방향 | 처치 자체보다 소화 효율과 정체 관리에 초점 |
| 라운드 후 성장 | 영양 자원으로 강화/해금/기관 업그레이드 |
| 실패 페널티 연계 | 미처리 음식 잔류(막힘) -> 구간 트러블 확률 증가 |

## 3) 타겟팅 규칙(기획)

| 타겟팅 | 설명 | 사용 상황 |
|---|---|---|
| First | 경로상 선두 음식 우선 | 출구 압박 감소 |
| Last | 후미 음식 우선 | 누적 압력 분산 |
| Cluster | 가장 많이 쌓인 군집 우선 | 정체/폭주 구간 안정화 |
| 속성 우선 | 지방/유제품 등 특정 속성 우선 | 상성 카운터 빌드 |

## 4) 현재 구현 상태 (2026-02-18 기준)

| 구분 | 상태 | 비고 |
|---|---|---|
| 타워 슬롯 렌더링 | 구현됨 | 테스트용 슬롯 원형 메시 렌더링 |
| 슬롯 클릭 감지 | 구현됨 | 클릭 시 UI 시트 연동 가능 구조 |
| 타워 상세 UI 갱신 | 구현됨 | `updateTowerInfo()`로 정보 반영 |
| 실제 타워 전투 로직 | 미구현(기획 단계) | 공격/상태이상/타겟팅 실전 로직 필요 |
| 타워 데이터 기반 정의 | 미구현(기획 단계) | JSON/설정 기반 확장 구조 필요 |

## 5) 구현 우선순위 제안

| 우선순위 | 작업 | 완료 기준 |
|---|---|---|
| P1 | 타워 데이터 스키마 정의 | 타입/사거리/속성/상태이상/코스트 명세 |
| P1 | 타워 매니저 시스템 추가 | 배치/타겟 선택/발사 주기 통합 관리 |
| P2 | 상태이상 파이프라인 구현 | 음식 엔티티에 분해/중화/유화/흡수 버프 적용 |
| P2 | 기관별 규칙 연동 | 위/소장/대장 구간 보정 적용 |
| P3 | 업그레이드/해금 UI 연결 | 라운드 보상과 성장 루프 완성 |
