### 2. 인게임 타워 성장 UI 요청

사용자가 메인 게임 화면에 타워 성장 정보를 실시간으로 표시하고 싶어함:

**요구사항:**
- 타워 상단: 별(⭐) 표시로 성급(1~7성) 시각화
- 타워 하단: 반원 게이지로 XP 진행률 표시
- 게이지 옆: 현재 레벨 표시 (Lv1~Lv12)

**이유:**
- 기존에는 타워를 클릭해서 상세 창을 열어야만 성장 정보 확인 가능
- 게임 중 한눈에 타워 상태를 파악하고 싶음
- 여러 타워의 성장 상태를 비교하고 싶음

### 3. EmojiRenderer에 성장 UI 구현

타워 렌더링 담당 클래스인 `EmojiRenderer.js`에 성장 정보 UI 추가:

**구현 내용:**

**1) 별 표시 (타워 상단):**
- 성급 수만큼 ⭐ 이모지 반복 표시
- 금색(#FFD700) + 검은색 그림자로 가독성 확보
- 타워 크기에 비례하여 별 크기 자동 조정
- 7개까지 가로로 배열

**2) 반원 게이지 (타워 하단):**
- Canvas 2D arc로 반원 그리기
- 배경: 어두운 회색 반투명
- 전경: XP 진행률에 따라 색상 변화
  - 0-33%: 초록색 (#4ade80)
  - 33-66%: 노란색 (#fbbf24)
  - 66-100%: 하늘색 (#60a5fa)
- 만랩 근처(90% 이상): 빛나는 효과(shadowBlur)

**3) 레벨 표시:**
- 초기 위치: 게이지 왼쪽 하단
- 일반: "Lv1", "Lv2", ... "Lv6"
- 만랩: "MAX" (금색)
- 검은 배경 박스로 가독성 확보

**기술 구조:**
```javascript
_drawTowerGrowthInfo(tower, x, y, size, scale) {
  // 1. 별 표시 (상단)
  for (let i = 0; i < tower.star; i++) {
    ctx.fillText('⭐', starX, starY);
  }

  // 2. 반원 게이지 (하단)
  ctx.arc(x, gaugeY, gaugeRadius, startAngle, endAngle);

  // 3. 레벨 표시
  ctx.fillText(levelText, levelX, levelY);
}
```

### 4. 게이지 방향 수정: 위쪽 반원 → 아래쪽 반원

첫 구현에서 반원이 위쪽(∩ 모양)으로 되어 있었음:

**문제:**
- 사용자: "반원 게이지가 아래쪽(U자 모양)이어야 한다"
- 게이지가 타워 위쪽에 표시되어 부자연스러움

**해결:**
```javascript
// Before: 위쪽 반원
ctx.arc(x, gaugeY, gaugeRadius, 0, Math.PI);

// After: 아래쪽 반원
ctx.arc(x, gaugeY, gaugeRadius, Math.PI, Math.PI * 2);
```

XP 진행 방향도 왼쪽 → 오른쪽으로 수정:
```javascript
const startAngle = Math.PI; // 왼쪽 (180도)
const endAngle = Math.PI + (Math.PI * xpPercent); // 오른쪽으로
```

### 6. XP 계산 방식 변경: 누적 → 레벨별 리셋

사용자가 XP 게이지가 레벨업 시 리셋되길 원함:

**기존 방식 (누적):**
- Lv1→Lv2: 0 → 20 XP (게이지 0% → 100%)
- Lv2→Lv3: 20 → 50 XP (게이지 40%에서 시작)
- 게이지가 중간부터 시작하여 혼란스러움

**변경 방식 (리셋):**
- Lv1→Lv2: 0 → 20 XP (게이지 0% → 100%)
- Lv2→Lv3: 0 → 30 XP (게이지 0% → 100%) ← 리셋
- 매 레벨업마다 게이지가 0%에서 시작

**구현:**
```javascript
_calculateXPPercent(tower, maxLevel) {
  // 현재 레벨과 다음 레벨의 XP 차이 계산
  const currentLevelXP = baseLevelCounts[prevLevelIndex] * multiplier;
  const nextLevelXP = baseLevelCounts[currentLevelIndex] * multiplier;

  const xpForThisLevel = nextLevelXP - currentLevelXP;
  const currentXPInLevel = tower.xp - currentLevelXP;

  return currentXPInLevel / xpForThisLevel;
}
```

**결과:**
- Lv1 (0~20 XP): 게이지 20 구간
- Lv2 (20~50 XP): 게이지 30 구간 ← 리셋
- Lv3 (50~100 XP): 게이지 50 구간 ← 리셋
- 레벨업마다 게이지가 0%에서 다시 채워짐

### 7. 최종 레이아웃 정리

사용자가 최종 시각적 레이아웃 정리:

**요구사항:**
1. 상단: 별
2. 중앙: 타워 emoji
3. 하단:
   - XP 반원 게이지 (타워 중심 기준 아래쪽)
   - 레벨 표시 (게이지 중앙, 타워 원의 정 하단)

**최종 구조:**
```
        ⭐⭐⭐
        (3성)

          🧪     ← 타워 emoji (중심)

       ◡━━━━━◡   ← XP 반원 게이지 (아래쪽 U자)

         Lv3     ← 레벨 표시 (게이지 중앙, 중심 정렬)
```

**스타일 개선:**
- 레벨 텍스트: 중앙 정렬 (`textAlign: "center"`)
- 그림자: 4px blur + 1px offset으로 가독성 극대화
- 폰트 크기: `size * 0.3` (타워 크기에 비례)
- 만랩: "MAX" 금색으로 강조

**7성 최대 제한:**
- 기존: 12성까지 (Lv5~Lv16)
- 변경: 7성까지 (Lv6~Lv12)
- 승급 비용: 14,000 NC + 150 SC (총합)

**2초 틱 XP 시스템:**
- 매 2초마다 1 XP 획득 (1800 XP/시간)

- 초반 가속형 곡선으로 빠른 진입
- 성장 가속 버프 시 +40% XP (SC 12 소비, 1시간)

**레벨 공식:**
- `Lmax = 5 + star`
- 1성: Lv6 (6 업그레이드 포인트)
- 7성: Lv12 (12 업그레이드 포인트)

**XP 요구량 (지수 곡선):**
- 1성 기준: [0, 20, 50, 100, 180, 300]
- 배율: `1.0 + (star - 1)^1.4 * 0.5`
- 고성급일수록 레벨당 필요 XP 증가

## 🤔 결정 사항 & 논의

### 1. 7성 vs 12성 체계

**선택지:**
- 12성: 긴 진행, 많은 콘텐츠
- 7성: 빠른 진행, 집중된 경험

**결정:** 7성
- 이유:
  - 사용자가 문서를 7성 기준으로 수정함
  - 모바일 방치형 게임에 적합한 길이
  - 10시간 진행 → 적당한 목표감
  - 승급 비용 부담 감소 (14,000 vs 49,500 NC)

### 2. XP 계산: 누적 vs 리셋

**선택지:**
- 누적: 계속 증가 (0→20→50→100...)
- 리셋: 레벨업 시 0으로 (0→20, 0→30, 0→50...)

**결정:** 리셋
- 이유:
  - 게이지가 항상 0%에서 시작하여 직관적
  - 레벨업 시 "완료감" 제공
  - 다른 게임들의 일반적인 방식
  - 사용자가 명시적으로 요청

### 3. 게이지 방향: 위쪽 vs 아래쪽

**선택지:**
- 위쪽(∩): 타워 위에 표시
- 아래쪽(U): 타워 아래에 표시

**결정:** 아래쪽(U자 모양)
- 이유:
  - 타워 emoji가 중심에 위치
  - 게이지가 하단에서 감싸는 느낌
  - 레벨 표시가 게이지 바로 아래 배치 가능
  - 시각적으로 안정감

### 4. 레벨 위치: 왼쪽 vs 중앙

**선택지:**
- 왼쪽: 게이지 왼쪽 하단
- 중앙: 게이지 중앙 하단

**결정:** 중앙
- 이유:
  - 타워 원의 정 하단에 위치하여 대칭적
  - 중앙 정렬로 가독성 향상
  - 시각적 균형감
  - 사용자가 명시적으로 요청

### 5. 효율성 링 제거

**이유:**
- 기존 SupplySystem 제거로 효율성 개념 사라짐
- 성장 정보 UI가 추가되어 링이 불필요
- 화면 정리로 가독성 향상

## 🐛 문제 & 해결

### 문제 1: XP 게이지가 중간부터 시작

**증상:**
- Lv2 타워의 게이지가 40%에서 시작
- 레벨업해도 게이지가 리셋되지 않음

**원인:**
- 누적 XP 방식으로 구현
- `tower.xp / requiredXP` 계산
- Lv2는 XP 20에서 시작, 다음 레벨 XP 50
- 20/50 = 40%로 표시됨

**해결:**
```javascript
// 현재 레벨과 다음 레벨의 XP 차이만 계산
const currentLevelXP = baseLevelCounts[prevLevelIndex] * multiplier;
const nextLevelXP = baseLevelCounts[currentLevelIndex] * multiplier;
const xpForThisLevel = nextLevelXP - currentLevelXP;
const currentXPInLevel = tower.xp - currentLevelXP;

return currentXPInLevel / xpForThisLevel;
```

**결과:** 매 레벨업마다 게이지 0%에서 시작

### 문제 2: 게이지 방향 반전

**증상:**
- 게이지가 위쪽(∩)으로 그려짐
- 사용자: "아래쪽(U자)이어야 한다"

**원인:**
- Arc 각도 잘못 설정
- `arc(x, y, r, 0, Math.PI)` → 위쪽 반원

**해결:**
```javascript
// 아래쪽 반원
arc(x, gaugeY, gaugeRadius, Math.PI, Math.PI * 2)

// 진행률도 왼쪽→오른쪽으로
const startAngle = Math.PI;
const endAngle = Math.PI + (Math.PI * xpPercent);
```

### 문제 3: 레벨 표시 가독성

**증상:**
- 레벨 텍스트가 게임 배경에 묻힘
- 특히 밝은 배경에서 보이지 않음

**시도 1: 검은 배경 박스**
```javascript
ctx.fillStyle = 'rgba(0, 0, 0, 0.7)';
ctx.fillRect(levelX - width/2, levelY - height/2, width, height);
```
- 문제: 박스가 눈에 거슬림

**시도 2: 그림자 효과**
```javascript
ctx.shadowColor = 'rgba(0, 0, 0, 0.8)';
ctx.shadowBlur = 4;
ctx.shadowOffsetX = 1;
ctx.shadowOffsetY = 1;
```
- 결과: 자연스럽고 가독성 좋음 ✓

**최종:** 그림자 방식 채택

## 💡 메모 / 인사이트

### Canvas 2D의 강력함

WebGL로 전환했지만, UI 요소는 Canvas 2D가 여전히 유용:
- **즉시 렌더링**: arc, fillText 등 간단한 API
- **동적 텍스트**: 레벨, 숫자 표시에 최적
- **그림자/효과**: shadowBlur로 쉬운 가독성 확보
- **이모지 지원**: fillText로 바로 그리기 가능

**교훈:** 올바른 도구를 올바른 곳에. WebGL은 대량 렌더링용, Canvas 2D는 UI용.

### 시각적 피드백의 중요성

인게임 성장 UI 추가 후:
- 타워 클릭 없이도 성장 상태 파악 가능
- 여러 타워 비교가 쉬워짐
- 게임 플레이 중 의사결정 향상

**효과:**
- "어느 타워가 승급 가능한가?" → 별 개수로 즉시 판단
- "어느 타워가 곧 레벨업하나?" → 게이지로 즉시 확인
- "이 타워 키울 가치 있나?" → 현재 상태로 빠른 판단

**교훈:** 정보를 숨기지 마라. 유저가 필요한 정보를 즉시 볼 수 있게 하라.

### XP 리셋의 심리학

누적 방식 vs 리셋 방식:

**누적 방식:**
- 수치상으로는 정확
- 하지만 게이지가 중간에서 시작하여 혼란스러움
- "왜 40%에서 시작하지?" → 유저 혼란

**리셋 방식:**
- 매번 0%에서 시작
- 100% 도달 시 레벨업의 "완료감"
- 다음 레벨을 위한 새로운 시작

**교훈:** 수학적 정확성보다 심리적 직관성이 중요. 유저는 감정으로 게임한다.

### 점진적 개선의 가치

이번 세션은 "검토"에서 시작:
1. 기존 구현 상태 확인
2. 작은 UI 개선 요청 (인게임 성장 정보)
3. 피드백 기반 반복 개선 (게이지 방향, 레벨 위치)
4. 세부 조정 (XP 계산, 가독성)

**각 단계마다:**
- 빠른 구현
- 사용자 피드백
- 즉시 수정
- 다음 개선으로

**교훈:** 큰 기능도 작은 조각으로 나누어 점진적으로 개선하라. 피드백 루프를 짧게 유지하라.

**효과:**

**교훈:** "대충 이렇게"보다 "정확히 이렇게"가 결과적으로 빠르다. 명확한 요구사항은 개발 속도를 높인다.
