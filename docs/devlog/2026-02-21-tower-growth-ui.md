## 🗣️ 작업 흐름

### 1. 타워 성장 시스템 구현 상태 점검

사용자가 게임을 플레이하며 타워 상세 UI가 미완성 상태임을 발견:

**문제점:**
- 타워 클릭 시 에러 발생 (`getEfficiencyMultiplier is not a function`)
- XP 진행 바 없음
- 성급(star) 정보 표시 부족
- 승급/리롤 버튼 없음
- 제거된 SupplySystem 관련 코드 잔재

**원인:**
- 이전 세션에서 재화 시스템(NC/SC)과 타워 성장 시스템은 구현했지만, UI가 따라오지 못함
- SupplySystem과 EfficiencyState 제거 후 UIController 업데이트 누락

### 2. 타워 상세 UI 완성

**구현 내용:**

**HTML 구조 추가:**
- 성장 정보 섹션 (성급 표시, XP 진행 바)
- 승급 버튼 (만랩 도달 시 표시, NC/SC 비용 표시)
- 스탯 리롤 버튼 (SC 20 소비)

**스탯 표시 개선:**
```
기존: 공격력 30
개선: 공격력 30 → 38.4 (+28%)
```
- 기본 스탯 → 승급 보너스 적용 스탯 (+증가율) 형태로 변경
- 유저가 성장 효과를 명확하게 확인 가능

**UIController 로직:**
- `_updateTowerGrowthInfo()`: XP 진행 바, 성급 표시 업데이트
- `_updateTowerActionButtons()`: 승급/리롤 버튼 상태 관리
- 만랩 도달 시에만 승급 버튼 표시
- 재화 부족 시 버튼 비활성화

**CSS 스타일:**
- XP 진행 바: 노란색 그라데이션, 반짝이는 애니메이션
- 성급 표시: 별 이모지 (최대 5개) + "N성" 텍스트
- 승급/리롤 버튼: Splatoon 스타일 (녹색/노란색)

### 3. SC 재화 UI 리뉴얼 요청

사용자가 SC 증가를 시각적으로 확인하고 싶어함:
- 기존: 텍스트로만 "⚡ 40/80 (50%)" 표시
- 요청: 좌우로 채워지는 게이지 바로 변경

**첫 번째 시도:**
- 숫자 뒤에 게이지 바 추가
- 하지만 사용자 의도와 다름: "숫자 안쪽에 게이지처럼 채워지게"

### 4. SC 게이지 바 디자인 개선

**두 번째 시도: 배경 채우기 방식**
- HTML 구조: 게이지 fill이 배경으로, 텍스트가 그 위에 표시
- CSS: `position: absolute`로 게이지와 텍스트 레이어링
- 결과: 숫자 배경이 왼쪽→오른쪽으로 채워지는 효과

**애니메이션 추가:**
- SC 증가 시 숫자가 커졌다 작아지는 펄스 효과
- 게이지 바가 빛나는 효과
- "+1" 파티클이 위로 올라가며 사라지는 효과

### 5. 테스트용 빠른 증가 속도

사용자 요청: "SC 증가를 3초마다 1씩 증가하게 해서 게이지 애니메이션을 확인하고 싶다"

**기존 설정:**
- 시간당 20 SC (3분마다 1씩)
- 너무 느려서 애니메이션 확인 어려움

**변경:**
- 시간당 180 SC (3초마다 1씩)
- 테스트용으로 60배 가속

**SC 증가 로직 개선:**
- 기존: 1초마다 체크하여 소수점 단위로 증가
- 개선: `scFractional`로 소수점 누적, 1이 되면 정수 1 증가
- 결과: SC가 정확히 1씩 증가하며 애니메이션 트리거

### 6. SC 게이지의 의미 명확화

사용자 피드백: "게이지가 전체 SC 총량(40/80)을 반영하지 말고, 다음 1 SC까지 차오르는 진행도만 표시해라"

**이해:**
- 전체 SC 총량은 이미 숫자로 표시됨 ("40/80")
- 게이지는 "다음 1 SC까지 얼마나 남았는지" 진행도를 보여주는 용도

**구현:**
- `sc-bar-base` 제거 (전체 총량 표시 제거)
- `sc-bar-progress`만 남김 (0~100% 진행도)
- `scFractional * 100`을 width로 설정
- 100% 도달 시 녹색으로 변경 후 리셋

**결과:**
- 게이지가 0%에서 시작하여 3초 동안 천천히 100%까지 차오름
- 100% 도달 시 SC +1, 녹색 펄스 효과, 게이지 리셋
- 유저가 SC 증가 속도를 직관적으로 확인 가능

### 7. 원래 밸런스로 복귀

사용자: "이제 게이지 모션이 잘 보이니까, 원래 설계대로 시간당 20 (3분마다 1씩)으로 되돌려라"

**최종 설정:**
- `passiveRegenPerHour: 20` (3분마다 1씩)
- 게이지가 3분 동안 천천히 차오름
- 게임의 원래 의도대로 작동

## 🎯 구현된 기능

### 1. 타워 상세 UI 완성

**성장 정보 표시:**
- **성급 표시**: ⭐⭐⭐ 3성 형태로 시각적 표시
- **XP 진행 바**: 현재 XP / 다음 레벨까지 필요 XP
  - 만렙 도달 시 "만렙" 표시
  - 노란색 그라데이션 + 반짝이는 애니메이션

**스탯 표시 개선:**
- 기본 스탯과 승급 보너스 적용 스탯을 함께 표시
- 증가율 퍼센트 표시
- 예: "공격력: 30 → 38.4 (+28%)"
- 상태성공률도 추가로 표시

**액션 버튼:**
- **승급 버튼**: 만랩 도달 시 표시, NC/SC 비용 명시
- **스탯 리롤 버튼**: SC 20 소비, 2성 이상부터 사용 가능
- 재화 부족 시 버튼 비활성화

### 2. SC 게이지 바 시스템

**게이지 의미:**
- 전체 SC 총량(40/80)은 숫자로 표시
- 게이지는 "다음 1 SC까지의 진행도 (0~100%)" 표시

**시각적 효과:**
- 게이지가 왼쪽→오른쪽으로 천천히 차오름
- 100% 도달 시:
  - 노란색 → 녹색으로 변경
  - 강한 펄스 + 글로우 효과
  - 숫자 증가 애니메이션
  - "+1" 파티클 위로 날아감
- 리셋되어 다시 0%부터 시작

**사용자 경험:**
- SC가 얼마나 빨리 차오르는지 실시간 확인 가능
- 3분 기다릴 때 지루하지 않음 (진행 상태 시각화)
- 게이지가 꽉 차는 순간의 만족감

### 3. 재화 시스템 최적화

**NC (영양 크레딧):**
- 숫자 카운트업 애니메이션 (300ms, 10단계)
- 부드러운 증가 효과

**SC (보급 차지):**
- 소수점 누적 방식 (`scFractional`)
- 정수 1이 될 때마다 실제 SC 증가
- UI 애니메이션 트리거 정확성 향상

## 🤔 결정 사항 & 논의

### 1. SC 게이지의 의미

**초기 구현:**
- 전체 SC 총량을 게이지로 표시 (40/80 = 50% 너비)
- 문제: 숫자와 게이지가 중복 정보

**최종 결정:**
- 게이지는 "다음 1 SC까지의 진행도"만 표시
- 이유:
  - 숫자로 총량 확인 가능 ("40/80")
  - 게이지는 증가 속도 체감용
  - 유저가 기다리는 동안 진행 상태 확인

### 2. 테스트용 빠른 속도

**원래 설계:** 3분마다 1씩 (시간당 20)
**테스트용:** 3초마다 1씩 (시간당 180)

**판단:**
- 개발 중에는 빠른 속도로 애니메이션 확인
- 최종적으로 원래 밸런스로 복귀
- 게이지 시스템 덕분에 3분도 지루하지 않음

### 3. 스탯 표시 방식

**선택지:**
1. 최종 스탯만 표시 (38.4)
2. 기본 → 최종 (+증가율) 표시 (30 → 38.4 +28%)

**결정:** 옵션 2 선택
- 이유:
  - 유저가 성장 효과를 명확하게 인지
  - 승급/리롤 전후 비교 가능
  - 게임의 핵심인 "성장"을 강조

## 🐛 문제 & 해결

### 1. SupplySystem 잔재 제거

**문제:**
- 타워 클릭 시 `getEfficiencyMultiplier is not a function` 에러
- 이유: UIController가 제거된 SupplySystem 메서드 호출

**해결:**
- `_setupSupplyButton()`: 빈 stub으로 변경
- `getEfficiencyMultiplier()` 호출 제거
- `efficiencyState` 대신 `star` (성급) 정보로 대체

### 2. SC 게이지 의미 혼동

**문제:**
- 처음에는 전체 SC 총량을 게이지로 표시
- 사용자 의도: 다음 1 SC까지의 진행도만 표시

**해결 과정:**
1. 시도 1: 전체 총량 게이지 (`scBarBase` + `scBarProgress`)
2. 사용자 피드백: "진행도만 보여줘"
3. 최종: `scBarBase` 제거, `scBarProgress`만 0~100%로 표시

**교훈:**
- 사용자 의도를 정확히 파악하는 것이 중요
- 프로토타입 빠르게 보여주고 피드백 받기

### 3. 애니메이션 확인의 어려움

**문제:**
- 원래 속도(3분마다 1씩)로는 애니메이션 테스트 어려움

**해결:**
- 설정을 `economyDefinitions.js`로 중앙화
- 테스트 시 빠른 속도로 변경 (시간당 180)
- 확인 후 원래 밸런스로 복귀 (시간당 20)

**개선점:**
- 향후 디버그 모드 추가 고려 (치트 키로 빠른 속도)

### 점진적 개선의 가치

**작업 순서:**
1. 로직 구현 (재화 시스템, 타워 성장)
2. UI 추가 (XP 바, 버튼)
3. 시각적 피드백 강화 (게이지 바, 애니메이션)

**인사이트:**

**economyDefinitions.js:**
