# 2026-02-22 개발 로그: 스킬트리 노드 카드 UI 개선

## 📋 세션 요약
- 스킬트리 업그레이드 노드 카드의 가독성을 집중 개선
- 노드 가로 20% 확장 → 레이아웃 재설계 → 설명 텍스트 공간 수학적 계산 후 폰트 최대화
- 태그 뱃지(`[지방]`, `[노출]` 등)를 뱃지 스타일로 변경해 어느 배경에서도 선명하게 표시

---

## 🗣️ 작업 흐름

### 1. 노드 가로 폭 20% 확장

사용자 요청: 스킬트리 노드를 가로로 20% 더 키우고 레이아웃도 글자가 잘 보이게 조정해달라. (실제 카드 HTML 스크린샷 첨부)

현황 파악:
- `UIController.js` 내 `_renderUpgradeTree()` 에서 `nodeWidth = 216` (이미 이전 세션에서 180→216으로 20% 올린 수치)
- 이번에 추가로 20% → `216 × 1.2 = 259.2` → **260px**
- 세로도 약간 늘림: 110 → **130px**

폰트 크기도 단계별 +1px씩 상향:
- 노드 이름: 12/13/14px → 13/14/15px
- 설명 텍스트: 10/11/12px → 11/12/13px (3단계 → 4단계로 세분화도 진행)

### 2. 레이아웃 재설계 — 헤더 상단 밀착, 설명 하단 배치

사용자 요청: 비용·이름을 카드 상단에 더 밀착하고, 설명 폰트를 잘림 없이 보이는 최대 크기로 카드 하단에 배치해달라.

변경 전 구조:
- content div: `justify-content: space-between` → 헤더와 설명 사이 공간이 카드 높이에 따라 자동 분산
- padding: 16px (상하 여백이 큼)

변경 후 구조:
- content div: `justify-content: flex-start` (헤더가 상단에 고정)
- 설명 `<p>`: `margin: auto 0 0 0` (남은 공간을 위쪽에 밀어넣어 설명이 하단 고정)
- padding: 16px → `10px 12px` (상하 여백 축소해 실제 콘텐츠 영역 확보)
- 뱃지(비용 표시) 높이: 36px → 28px, border 4px → 3px로 슬림화

### 3. 설명 텍스트 공간 수학적 재계산

사용자 요청: 여유 공간을 다시 계산해서 더 키워달라.

LLM이 직접 계산:

| 항목 | 계산 |
|---|---|
| 카드 전체 높이 | 130px (border 포함, `box-sizing: border-box`) |
| 내부 높이 | 130 - border(10) - padding(20) = **100px** |
| 내부 너비 | 260 - border(10) - padding(24) = **226px** |
| 헤더 행 높이 | 28px (뱃지 기준) |
| 설명 상단 여백 | 4px |
| **설명 가용 높이** | **68px** |

줄간격 1.4 기준 폰트별 수용 가능 글자수:

| 폰트 | 줄 높이 | 최대 줄수 | 줄당 한글 | 총 수용 |
|---|---|---|---|---|
| 15px | 21px | 3.2줄 | ~15자 | ~48자 |
| 14px | 19.6px | 3.5줄 | ~16자 | ~56자 |
| 13px | 18.2px | 3.7줄 | ~17자 | ~63자 |
| 12px | 16.8px | 4.0줄 | ~18자 | ~72자 |

이를 기반으로 폰트 티어 재설정:
- ≤ 30자 → 15px / 31~45자 → 14px / 46~60자 → 13px / > 60자 → 12px

### 4. 전 티어 +3px 추가 상향

사용자 요청: 3px씩 더 키워달라.

별도 재계산 없이 요청대로 각 티어에 +3px 일괄 적용:
- ≤ 30자: 15 → **18px**
- 31~45자: 14 → **17px**
- 46~60자: 13 → **16px**
- > 60자: 12 → **15px**

카드에 `overflow: visible` 이 설정되어 있어 텍스트가 약간 넘쳐도 잘림(클립)이 발생하지 않음.

### 5. 태그 뱃지 가독성 개선 — 배경색 적용

사용자 요청: 노드 카드 배경색을 고려해서 태그 폰트 컬러를 더 잘 보이게 수정해달라. 이후 추가로 글자 배경색도 적용 가능한지 질문.

문제 분석:
- 카드 배경이 3가지: 흰색(활성화 가능), 연회색(비활성), 청록 그라디언트(활성)
- 기존 태그 색상 중 다수(`#f5c842` 노랑, `#80d8ff` 하늘, `#40e0ff` 청록 등)가 흰색·청록 배경에서 거의 안 보임

해결 방향: 글자 색만 바꾸는 것은 배경마다 한계가 있으므로 **뱃지 스타일**로 전환.
- 각 태그 색상을 흰 텍스트가 잘 보이는 짙은 버전으로 재조정
- `<span>`에 배경색 + 흰 텍스트 + 얇은 테두리 + 패딩 + border-radius 적용
- 사용자가 배경색 적용 가능 여부를 묻기 전에 이미 뱃지 방식으로 구현 완료

## 🎯 구현된 기능

### 더 넓고 높아진 스킬트리 노드 카드

**무엇인가:** 업그레이드 트리의 각 노드 카드가 가로로 20% 더 넓어지고(216→260px) 세로도 약간 높아졌다(110→130px).

**왜 필요했나:** 기존 카드가 좁아서 설명 텍스트가 작은 폰트로 눌려 보이고, 한 눈에 내용이 들어오지 않았다.

### 헤더-설명 분리 레이아웃

**무엇인가:** 비용(🍎 숫자)과 노드 이름이 카드 맨 위에 밀착 배치되고, 효과 설명은 카드 아랫부분에 따로 자리 잡는다.

**왜 필요했나:** 기존에는 헤더와 설명이 카드 높이에 따라 중간에 균등 분산되어 어수선하게 보였다.

### 설명 텍스트 최대 폰트 자동 계산

**무엇인가:** 설명 텍스트 길이에 따라 4단계(18/17/16/15px)로 폰트 크기가 자동 결정된다. 카드 여유 공간(68px)과 줄 너비(226px)를 실제로 계산한 수치 기반.

**왜 필요했나:** "가능한 한 크게, 하지만 잘리지 않게"라는 요구를 임의의 수치가 아닌 근거 있는 최대값으로 구현하기 위해.

### 태그 뱃지 스타일

**무엇인가:** `[지방]`, `[노출]` 같은 태그가 이제 각 의미에 맞는 색상 배경의 작은 뱃지 형태로 표시된다. 흰색, 회색, 청록색 카드 배경 모두에서 선명하게 보인다.

**왜 필요했나:** 글자 색만으로는 밝은 배경과 청록 활성 배경 양쪽에서 동시에 잘 보이는 색을 찾기 어렵다. 배경색이 있는 뱃지 방식이 어느 배경에서도 일관된 가독성을 보장한다.

## 🤔 결정 사항 & 논의

### 설명 폰트 티어 계산 방식

기존에는 "글자 수 25/40/55" 같은 임의 구간에 임의 폰트 크기를 썼다. 이번 세션에서 카드 실제 내부 크기를 직접 계산(border + padding 공제)하고, 줄간격·한글 문자 폭 추정치를 반영해 각 폰트 크기가 실제로 몇 글자까지 수용 가능한지 수치로 검증한 후 티어 경계를 정했다.

### 뱃지 텍스트 색상

모든 뱃지에 흰색 텍스트를 쓰기 위해 기존 색상(주로 밝은 계열)을 어두운 버전으로 교체했다.
예: 탄수화물 `#f5c842(밝은 노랑)` → `#b08000(짙은 황색)`, 탄산음료 `#40e0ff(밝은 청록)` → `#0095aa(짙은 청록)`.
원래 색조의 "느낌"은 유지하면서 흰 텍스트 가독성을 확보하는 방향.

### 특별한 버그 없음

모든 변경이 단일 파일(`UIController.js`, `TagFormatter.js`) 수정으로 완결됐고, 기존 기능과 충돌하는 부분 없음. 카드 `overflow: visible` 설정 덕분에 폰트를 공격적으로 키워도 텍스트 클립 걱정이 없었다.

## 💡 메모 / 인사이트

### 레이아웃 여백 계산의 중요성

"더 크게"라는 막연한 요청에 답하는 가장 확실한 방법은 실제 가용 공간을 계산하는 것이었다. border, padding, 헤더 높이를 모두 공제한 68px라는 수치가 나오자, 폰트별 수용 가능 글자수를 표로 정리하고 경계값을 근거 있게 정할 수 있었다. 임의 수치보다 설득력 있고 나중에 카드 크기가 바뀌어도 같은 방식으로 재계산할 수 있다.

### 뱃지 스타일의 배경색 독립성

글자 색 하나로 "흰 배경, 회색 배경, 청록 배경" 세 가지를 동시에 만족시키는 것은 물리적으로 불가능(밝은 배경엔 어두운 색, 어두운 배경엔 밝은 색이 필요하기 때문). 배경색 있는 뱃지는 자체 배경으로 주변 카드 색과 격리되므로 이 제약에서 자유롭다. 여러 배경 위에서 동작해야 하는 강조 텍스트에 적합한 패턴.
