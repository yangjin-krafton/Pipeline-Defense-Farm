# Devlog: 경로 편집기 개발 도구

**Date:** 2026-02-18

## Summary

게임 내 음식 이동 경로(PATH_POINTS)를 시각적으로 편집할 수 있는 개발용 편집기 구현. 드래그 앤 드롭으로 포인트 이동, 클릭으로 추가/삭제, 그리드 스냅 기능, config.js 직접 저장 기능 포함.

---

## 🗣️ 작업 흐름

### 1. 초기 요청: 경로 편집기 필요

사용자가 PATH_POINTS의 위치와 개수를 쉽게 바꿀 수 있는 **개발용 편집기**를 요청:

**요구사항:**
- 2D XY 좌표 시스템
- 드래그로 포인트 이동
- 추가/삭제 가능
- 그리드 스냅 기능

**배경:** 현재는 `config.js` 파일에서 직접 좌표를 수정해야 하는데, 이는:
- 좌표를 상상하기 어려움
- 시행착오가 많음
- 실시간 미리보기 불가

### 2. PathEditor 클래스 설계

경로 편집을 위한 전용 클래스를 설계:

**핵심 기능:**
- **드래그 앤 드롭**: 포인트를 마우스로 잡고 이동
- **추가**: 빈 공간 클릭 시 새 포인트 생성
- **삭제**: 우클릭 또는 Delete 키
- **그리드 스냅**: 10px 단위로 자동 정렬
- **실시간 미리보기**: 편집 중 경로가 바로 업데이트

**시각적 요소:**
- 노란색 경로선
- 포인트마다 번호 표시
- 호버 시 하이라이트
- 드래그 중인 포인트는 빨간색
- 그리드선 표시 (토글 가능)

### 3. 문제 발견: 드래그가 작동하지 않음

편집기를 통합한 후, 사용자가 드래그가 작동하지 않는다고 보고:

**증상:**
- 마우스 클릭 이벤트는 작동
- 드래그 이벤트는 무시됨
- 콘솔 로그에 container 클릭만 출력

**원인 파악:**
- `emojiCanvas`의 CSS에 `pointer-events: none` 설정
- 게임 중에는 이모지 레이어가 클릭을 받으면 안 되지만
- 편집 모드에서는 반대로 이벤트를 받아야 함

**해결책:**
- 편집 모드 활성화 시: `pointer-events: auto` + `z-index: 1000`
- 편집 모드 비활성화 시: `pointer-events: none` 복원

### 4. config.js 저장 기능 추가

사용자가 편집 후 실제 `config.js` 파일에 저장하는 방법을 문의:

**1차 구현: 클립보드 복사**
- E 키: JSON 형식으로 콘솔 출력 + 클립보드 복사
- 사용자가 수동으로 config.js에 붙여넣기

**문제점:** 매번 복사/붙여넣기가 번거로움

**2차 구현: 파일 저장 기능**
- S 키: File System Access API로 직접 파일 저장
- 저장 버튼: UI에서 클릭으로 저장
- config.js 전체 내용을 자동 생성

**동작:**
1. S 키 또는 저장 버튼 클릭
2. 브라우저 파일 선택 다이얼로그 열림
3. `src/js/config.js` 선택
4. 편집된 PATH_POINTS로 파일 덮어쓰기
5. 성공 알림 표시

**호환성:**
- Chrome/Edge: File System Access API 사용 (직접 저장)
- 기타 브라우저: 다운로드로 자동 전환

## 🎯 구현된 기능

### 1. 비주얼 경로 편집기

**활성화:**
- **Ctrl+Shift+E**: 편집 모드 토글
- 편집 모드 진입 시 게임 일시정지
- 캔버스 위에 편집 UI 오버레이

**편집 기능:**
- **왼쪽 클릭 (빈 공간)**: 새 포인트 추가
- **왼쪽 클릭 + 드래그 (포인트)**: 위치 이동
- **오른쪽 클릭 (포인트)**: 삭제
- **Delete 키**: 호버된 포인트 삭제
- **G 키**: 그리드 스냅 ON/OFF

**시각화:**
- 노란색 경로선으로 포인트 연결
- 포인트마다 0부터 시작하는 인덱스 표시
- 선택/호버 상태 시각적 피드백
- 10px 그리드선 (토글 가능)

### 2. 실시간 미리보기

편집하는 동안 게임 화면의 경로가 실시간으로 업데이트:

**작동 원리:**
- PathEditor가 포인트 변경을 감지
- 콜백으로 변경 사항 전달
- WebGL 메시 재생성
- 화면에 새 경로 즉시 반영

**장점:**
- 변경 사항을 바로 확인
- 시행착오 시간 단축
- 직관적인 편집 경험

### 3. 그리드 스냅 시스템

포인트를 그리드에 자동으로 정렬:

**설정:**
- 기본 그리드 크기: 10px
- G 키로 ON/OFF 토글
- 그리드선 시각적 표시

**효과:**
- 포인트가 깔끔하게 정렬됨
- 수치가 10의 배수로 떨어짐
- 나중에 조정하기 쉬움

### 4. 파일 저장 기능

편집한 경로를 config.js에 직접 저장:

**방법 1: 키보드 단축키**
- **S 키**: 파일 저장 다이얼로그 열기
- 경로 선택 후 저장
- 성공 알림 팝업

**방법 2: UI 버튼**
- 화면 우측 상단에 "💾 Save to config.js" 버튼
- 클릭으로 저장 다이얼로그 실행

**방법 3: 클립보드 복사**
- **E 키**: JSON 형식으로 복사
- 수동으로 config.js에 붙여넣기

**파일 내용:**
```javascript
export const PATH_POINTS = [
  { x: 180, y: 0 },
  { x: 180, y: 40 },
  // ... 편집된 포인트들
];
```

### 5. 개발자 UI

편집 모드 활성화 시 화면에 표시되는 정보:

**캔버스 상단 정보 패널:**
- PATH EDITOR (Dev Mode) 제목
- 현재 포인트 개수
- 그리드 상태 (ON/OFF)
- 단축키 안내

**화면 우측 상단 메뉴:**
- 컨트롤 단축키 목록
- 마우스 조작 안내
- 저장 버튼
- 세련된 디자인 (검정 배경 + 청록 테두리)

**알림 시스템:**
- 중앙에 페이드 인/아웃 알림
- 저장 성공 시: "Config file saved successfully!"
- 복사 성공 시: "Path copied to clipboard!"

## 🤔 결정 사항 & 논의

### 1. 편집 모드 vs 항상 활성화

**고려사항:**
- 항상 활성화: 편리하지만 게임 플레이 방해
- 편집 모드: 토글 필요하지만 깔끔함

**결정:** 편집 모드 (Ctrl+Shift+E 토글)
- 게임 플레이와 편집을 명확히 분리
- 개발 중에만 필요한 기능
- 실수로 경로를 변경할 위험 없음

### 2. 그리드 크기: 10px vs 다른 값

**고려사항:**
- 5px: 더 세밀하지만 숫자가 복잡
- 10px: 깔끔한 숫자, 적당한 정밀도
- 20px: 너무 큰 간격

**결정:** 10px
- 가상 좌표계 (360x640)에서 적절한 크기
- 계산하기 쉬운 숫자
- 필요시 그리드를 끄고 자유롭게 배치 가능

### 3. 저장 방식: 다운로드 vs 직접 저장

**다운로드 방식:**
- 장점: 모든 브라우저 지원
- 단점: 매번 파일을 이동/교체해야 함

**직접 저장 (File System Access API):**
- 장점: 한 번에 파일 덮어쓰기
- 단점: Chrome/Edge만 지원

**결정:** 둘 다 구현
- 기본: File System Access API 시도
- 폴백: 지원 안 하면 자동으로 다운로드 방식

### 4. 최소 포인트 개수: 2개

**이유:**
- 경로는 최소 시작점과 끝점 필요
- 2개 미만이면 경로를 그릴 수 없음
- 삭제 시 2개 이하로 내려가면 경고 표시

## 🐛 문제 & 해결

### 문제 1: 드래그 이벤트가 발생하지 않음

**증상:**
- 포인트를 클릭해도 드래그되지 않음
- 컨테이너 클릭 이벤트만 계속 발생
- 콘솔에 좌표는 찍히지만 PathEditor 이벤트는 무시됨

**원인:**
- `emojiCanvas`에 `pointer-events: none` CSS 설정
- 게임 플레이 중 이모지가 클릭 이벤트를 막지 않도록 설정된 것
- 편집 모드에서는 반대로 이벤트를 받아야 함

**해결:**
1. 편집 모드 활성화 시:
   - `canvas.style.pointerEvents = 'auto'`
   - `canvas.style.zIndex = '1000'`
2. 편집 모드 비활성화 시:
   - `canvas.style.pointerEvents = 'none'` 복원
   - `canvas.style.zIndex = ''` 복원

**결과:** 드래그가 정상 작동

### 문제 2: 타워 슬롯 클릭과 충돌

**증상:**
- 편집 모드에서도 타워 슬롯 클릭 이벤트가 발생
- 편집 중에 실수로 타워 메뉴가 열림

**원인:**
- 타워 슬롯 클릭 리스너가 항상 활성화
- 편집 모드 여부를 확인하지 않음

**해결:**
```javascript
if (window.isPathEditorActive && window.isPathEditorActive()) {
  return; // 편집 모드면 타워 클릭 무시
}
```

**결과:** 편집 모드에서 타워 클릭 비활성화

### 문제 3: 파일 저장 후 변경사항 미반영

**증상:**
- config.js 파일을 저장했지만
- 브라우저에서 새로고침해도 이전 경로가 표시됨

**원인:**
- 브라우저 모듈 캐싱
- ES6 모듈은 한 번 로드되면 캐시됨

**해결:**
- 사용자에게 "새로고침 필요" 안내
- Ctrl+Shift+R (캐시 무시 새로고침) 권장
- 또는 개발자 도구에서 "Disable cache" 활성화

**결과:** 저장 후 정상 반영

## 💡 기술 노트

### File System Access API

Chrome/Edge에서 사용 가능한 최신 파일 API:

**기능:**
- 사용자가 선택한 파일에 직접 쓰기
- 파일 선택 다이얼로그 표시
- 보안: 사용자가 명시적으로 허용해야 함

**구현:**
```javascript
const handle = await window.showSaveFilePicker(options);
const writable = await handle.createWritable();
await writable.write(content);
await writable.close();
```

**폴백:**
```javascript
if (!window.showSaveFilePicker) {
  // Blob + 다운로드 링크로 대체
  const blob = new Blob([content]);
  const url = URL.createObjectURL(blob);
  // a.download = 'config.js'
}
```

### Canvas 좌표 변환

캔버스 크기와 가상 좌표계 변환:

**문제:**
- 캔버스 CSS 크기: 가변 (화면에 맞춤)
- 가상 좌표계: 고정 (360x640)
- 마우스 이벤트: CSS 크기 기준

**변환:**
```javascript
const rect = canvas.getBoundingClientRect();
const x = e.clientX - rect.left;
const virtualX = (x / rect.width) * VIRTUAL_W;
```

### 실시간 메시 업데이트

경로 변경 시 WebGL 메시를 재생성:

**흐름:**
1. PathEditor에서 포인트 변경
2. `onPathChanged` 콜백 호출
3. `buildPolylineMesh(newPoints)` 재계산
4. 기존 메시 버퍼 삭제
5. 새 메시 생성 및 업로드
6. 다음 프레임에 새 경로 렌더링

**성능:**
- 편집 중에는 게임이 일시정지되어 있어 부담 없음
- 메시 재생성은 밀리초 단위로 빠름

## 💡 메모 / 인사이트

### 개발 도구의 가치

이번에 경로 편집기를 만들면서 **개발 도구의 중요성**을 실감했다:

**이전 방식:**
- config.js에서 수치 입력
- 결과 확인하려면 새로고침
- 좌표를 상상으로 추측
- 10번 이상 수정 반복

**편집기 사용:**
- 시각적으로 편집
- 실시간 미리보기
- 직관적인 드래그 앤 드롭
- 한 번에 원하는 결과 달성

좋은 개발 도구는 **생산성을 10배 높인다**.

### 모드 분리의 중요성

게임 모드와 편집 모드를 명확히 분리한 것이 좋은 선택이었다:

**장점:**
- 실수로 경로 변경할 위험 없음
- 각 모드가 독립적으로 동작
- UI가 깔끔함 (항상 표시되지 않음)

**단점:**
- 토글 키 조합을 기억해야 함 (Ctrl+Shift+E)

하지만 장점이 단점을 압도한다.

### File System Access API의 힘

브라우저에서 직접 파일을 수정할 수 있게 된 것은 **게임 체인저**:

**이전:**
- 복사 → 에디터 열기 → 붙여넣기 → 저장
- 4단계 작업

**현재:**
- S 키 → 파일 선택 → 저장
- 2단계 작업

웹 앱이 점점 네이티브 앱에 가까워지고 있다.

### 즉각적인 피드백

편집 중 경로가 실시간으로 업데이트되는 것이 핵심:

- **지연된 피드백**: 변경 → 저장 → 새로고침 → 확인 (10초)
- **즉각적 피드백**: 변경 → 확인 (0.1초)

100배 빠른 피드백 루프는 창의성을 폭발시킨다.

### 그리드 스냅의 양날의 검

그리드 스냅은 편리하지만 때로는 방해가 된다:

**해결책:**
- 기본적으로 ON (대부분의 경우 유용)
- G 키로 쉽게 토글 (세밀한 조정 필요 시)

옵션을 제공하되, 좋은 기본값을 선택하는 것이 중요하다.
