## 🗣️ 작업 흐름

### 1. 개발자 명령어: 타워 경험치 추가

승급 시스템 테스트를 위한 콘솔 명령어 구현 요청:

**요구사항:**
- 선택한 타워에 경험치 추가하는 명령어
- `addXP(500)` 형태로 간단하게 사용 가능
- 만렙까지 빠르게 올리는 명령어도 필요

**구현 내용:**
- `addXP(amount)`: 선택한 타워에 경험치 추가 (기본값 100)
- `maxLevel()`: 현재 성급의 만렙까지 즉시 레벨업
- `towerInfo()`: 선택한 타워의 현재 상태 확인 (성급, 레벨, XP, 승급 가능 여부)
- Before/After 정보 출력으로 변화 확인 가능

**결과:**
콘솔에서 `maxLevel()` 입력 → 타워가 즉시 만렙 → 승급 조건 테스트 가능

### 5. 승급 비용 밸런싱 재조정

사용자 피드백: "1성 → 2성 비용 1000 NC가 너무 비싸다"

**문제점 분석:**
- 일일 NC 수급: 약 2,720 NC/day
- 타워 설치 비용: 150-180 NC
- 1→2성 비용: 1,000 NC (일일 수급의 37%)
- 타워 설치 + 승급 = 거의 반나절 수급량

**개선안:**
```
기존 → 개선 (감소율)
1→2성: 1,000 → 200 NC (↓80%)
2→3성: 1,400 → 400 NC (↓71%)
3→4성: 1,900 → 700 NC (↓63%)
4→5성: 2,500 → 1,200 NC (↓52%)
5→6성: 3,200 → 2,000 NC (↓38%)
6→7성: 4,000 → 3,500 NC (↓13%)
```

**설계 철학:**
- 초반 (1-3성): 빠른 진입 (첫날부터 여러 승급 가능)
- 중반 (4-5성): 적당한 도전 (1-2일 모아야 함)
- 후반 (6-7성): 높은 도전 (3일 정도)

**결과:**
- Day 1: 13개 타워 1→2성 승급 가능
- 초반 진입장벽 대폭 완화
- 후반으로 갈수록 도전적

### 7. 영구 각인 시스템 구현

문서의 "영구 스킬트리 각인" 규칙 구현:

**핵심 개념:**
- 승급 시 활성화된 스킬트리 노드 중 하나를 영구 각인으로 선택
- 각인된 노드의 능력이 영구히 유지됨
- 다음 승급 시에도 같은 노드를 또 각인 가능 (최대 7회)
- 각인 횟수에 따라 스킬트리 활성화 비용 지수 증가

**구현 1: 각인 데이터 구조**

```javascript
// BaseTower.js
this.imprints = [];  // 각인 목록
this.imprintCounts = new Map();  // 노드별 각인 횟수
```

각인 저장:
```javascript
tower.imprints.push({
  nodeNumber: 1,
  nodeName: "풀피 집중 조준",
  nodeDescription: "기본 피해 +8%, HP 95%+ 적 추가 피해 +12%",
  acquiredStar: 2,  // 2성 승급 시 획득
  imprintedNode: originalNode  // UpgradeNode 참조 (모듈 포함)
});

tower.imprintCounts.set(1, 1);  // 노드1을 1회 각인
```

**구현 2: 각인 효과 적용**

`UpgradeNode.js`의 `getActiveModulesByType()` 수정:

```javascript
// 1. 현재 활성화된 노드의 모듈
for (const node of this.activeNodes) {
  modules.push(...node.modules);
}

// 2. 각인된 노드의 모듈 (영구 효과) ← 새로 추가!
if (this.tower && this.tower.imprints) {
  for (const imprint of this.tower.imprints) {
    if (imprint.imprintedNode) {
      modules.push(...imprint.imprintedNode.modules);
    }
  }
}
```

**결과:**
- 스킬트리가 리셋되어도 각인된 능력은 유지
- 각인 3회 → 모듈이 3번 적용됨 (효과 중첩!)

**구현 3: 각인 중복 및 비용 증가**

각인 횟수에 따른 비용 배율:
```javascript
// 지수 곡선: multiplier = 1.0 + (count ^ 1.4) * 0.5
0회: x1.0
1회: x1.5 (+50%)
2회: x2.2 (+122%)
3회: x3.2 (+222%)
7회: x11.5 (+1050%)
```

스킬트리 카드 표시:
```
노드 이름 +N (각인 횟수)
"연속 집중 공격 +2" (2회 각인)
비용: 18 NC → 40 NC (x2.2)
```

### 9. 스킬트리 능력 리뉴얼

사용자 요청: "각인 중복을 고려해서 모든 노드를 수치 증폭 형태로 변경해라"

**문제점:**
- 타겟팅 변경 노드 (priority: 'first') → 각인 중복 시 의미 없음
- 즉시 처치 노드 (execute) → 각인 중복 시 동작 불명확
- 최소 피해 보장 → 각인 중복 시 어떻게 계산?

**해결: 모든 노드를 수치 증폭으로 변경**

**노드 1: "선두 적 포커스" → "풀피 집중 조준"**
```javascript
// 변경 전
TargetingModule({ priority: 'first' })  // 각인 중복 X

// 변경 후
DamageModule({ damageMultiplier: 1.08 })  // 기본 +8%
TriggerModule({
  condition: hp >= 95%,
  effect: { damageBonus: 0.12 }  // 풀피 +12%
})
// 각인 3회: 기본 ×1.26, 풀피 +36% ✅
```

**노드 6: "고위협 관통 조준" → "중장갑 관통 사격"**
```javascript
// 변경 전
TargetingModule({ priority: 'threat' })  // 직관적이지 않음
effect: 'threat 15+ 대상 피해 +10%'

// 변경 후
TriggerModule({
  condition: armor >= 10,  // 방어력으로 변경
  effect: { damageBonus: 0.15 }  // +15%
})
effect: '방어력 10+ 적 추가 피해 +15%'
```

**노드 8: "임계 안정화" → "안정 화력 증폭"**
```javascript
// 변경 전
SafetyModule({ minDamageGuarantee: 0.3 })  // 각인 중복 시 어떻게?

// 변경 후
DamageModule({
  damageMultiplier: 1.12,  // 피해 +12%
  attackSpeedMultiplier: 1.05  // 공속 +5%
})
// 각인 3회: 피해 ×1.40, 공속 ×1.16 ✅
```

**노드 11: "임계 처형 알고리즘" → "임계 폭딜 알고리즘"**
```javascript
// 변경 전
TriggerEffect.execute()  // 즉시 처치 - 각인 중복 시?

// 변경 후
TriggerModule({
  condition: hp <= 35%,
  effect: {
    damageBonus: 0.40,  // 피해 +40%
    critChanceBonus: 0.15  // 치명타 +15%p
  }
})
// 각인 3회: 피해 +120%, 치명타 +45%p ✅
```

### 10. 각인 중복 표시 시스템

사용자 요청: "각인된 노드는 '+1', '+2' 형태로 표시하고, 각인 보너스를 효과 텍스트에 직접 표시해라"

**구현 1: 스킬트리 노드 이름에 +N 표시**

```javascript
const imprintCount = tower.imprintCounts.get(node.nodeNumber) || 0;
const imprintSuffix = imprintCount > 0
  ? `<span style="color: #ffd700;">+${imprintCount}</span>`
  : '';

노드 이름: "연속 집중 공격 +2"  (금색으로 강조)
```

**구현 2: 효과 텍스트에 각인 보너스 계산**

`_getEnhancedEffectText()` 함수로 동적 생성:

```javascript
// 원본 effect:
"기본 피해 +6%, 같은 적 연속 공격 시 추가 피해 +12%"

// 각인 2회 시 변환:
"기본 피해 +6% (+12.4%), 같은 적 연속 공격 시 추가 피해 +12% (+24%)"
```

**계산 로직:**
- "기본 피해 +N%": 곱연산 (1.06^count)
- "추가 피해 +N%": 가산 (base × count)
- "공격속도 +N%": 곱연산
- "치명타 확률 +N%p": 가산
- "치명타 배율 +N.N": 가산

**각 노드에 중복 효과 주석 추가:**
```javascript
// 1. 풀피 집중 조준 (DM+TR)
// 각인 중복 시: 기본 피해 곱연산(×1.08^n), 풀피 보너스 가산(+12%×n)
new UpgradeNode({
  modules: [
    DamageModule({ damageMultiplier: 1.08 }),  // [각인 중복: 곱연산]
    TriggerModule({ damageBonus: 0.12 })  // [각인 중복: 가산]
  ]
})
```

**중복 제거 문제 해결:**

초기 구현에서 보너스가 2번 표시되는 버그 발생:
```
기본 피해 +10% (+10%) (+10.0%)  ← 중복!
```

**원인:**
- 정규식 패턴이 겹침
- "기본 피해 +10%" 처리 후 → "피해 +10%" 패턴이 또 매칭됨

**해결:**
- 마커 방식 시도 (`___PROCESSED___`) → Negative lookbehind 환경 호환성 문제
- 최종: 매칭 수집 후 역순 교체 방식
  1. 모든 매칭을 배열로 수집
  2. 역순으로 교체 (인덱스 유지)
  3. 각 매칭이 정확히 1번만 처리됨

- 타워 기본 정보와 각인이 하나의 블록에
- 시각적으로 더 일관성 있음
- 스킬트리와 명확히 분리

### 13. 설명 문구 한글화

사용자 요청: "expose, shock, mark 같은 영문 타입을 설명에서는 한글로 표시해라"

**원칙:**
- **코드 내부**: 영문 유지 (시스템 안정성)
  ```javascript
  e.type === 'expose'  // ✅
  statusType: 'mark'   // ✅
  ```

- **사용자 표시**: 한글 사용 (직관성)
  ```javascript
  effect: '취약 대상 피해 +12%'  // ✅ (코드: expose)
  effect: '감전 대상 치명타 +10%p'  // ✅ (코드: shock)
  ```

- `shock` → 감전

- `corrode` → 부식

## 🤔 결정 사항 & 논의

### 1. 승급 비용: 높은 진입장벽 vs 빠른 진입

**초기 설정 (문서):**
- 1→2성: 1,000 NC
- 일일 수급의 37%
- 첫날에 1-2개 타워만 승급 가능

**문제:**
- 진입장벽이 너무 높음
- 유저가 승급 시스템을 빨리 체험하지 못함
- 초반 재미 부족

**재조정 결정:**
- 1→2성: 200 NC (80% 감소)
- 첫날에 10개 이상 타워 승급 가능
- 초반 빠른 진입, 후반 도전적

**이유:**
- 방치형 게임은 초반 보상이 중요
- 승급 시스템을 빨리 체험해야 재미를 느낌
- 후반 비용을 높여서 장기 목표 제공

### 2. 각인 시스템: 고정 능력 vs 중복 가능

**고려사항:**
- **고정**: 한 번 각인하면 끝 (간단)
- **중복**: 같은 노드 여러 번 각인 (복잡하지만 깊이 있음)

**결정:** 중복 각인 허용 (최대 7회)

**이유:**
- 문서에서 "동일 계열 중첩 허용" 명시
- 특화 빌드 가능 (연속 공격에 올인)
- 장기 성장 목표 제공
- 비용 지수 증가로 밸런스 유지

### 3. 각인 보너스 표시: 별도 박스 vs 텍스트 내부

**초기 구현:**
```
연속 집중 공격 +2
기본 피해 +6%, 추가 피해 +12%
[별도 박스] ✨ 각인 2회 보유 (비용 x2.2)
```

**사용자 요청:**
- 별도 박스 제거
- 효과 텍스트에 직접 보너스 표시

**최종:**
```
연속 집중 공격 +2
기본 피해 +6% (+12.4%), 추가 피해 +12% (+24%)
```

**이유:**
- 정보 밀도 증가
- 각 수치의 실제 효과를 즉시 확인 가능
- 별도 박스가 공간 낭비

### 4. 스킬트리 노드 설계: 특수 효과 vs 수치 증폭

**문제:**
- 타겟팅 변경 (priority: 'first') → 각인 중복 시 의미 없음
- 즉시 처치 (execute) → 각인 중복 시 동작 불명확

**결정:** 모든 노드를 수치 증폭으로 통일

**이유:**
- 각인 중복 시 명확한 효과
- 밸런싱이 수치로 가능
- 유저가 성장을 체감 가능
- 특수 효과는 수치 증폭으로 대체 가능
  - "타겟팅 변경" → "특정 조건 피해 증폭"
  - "즉시 처치" → "초고피해 증폭"

## 🐛 문제 & 해결

### 문제 1: 각인 보너스 중복 표시

**증상:**
```html
기본 피해 +10% (+10%) (+10.0%)
추가 피해 +15% (+15%) (+15%)
```

**원인:**
- 정규식 패턴이 겹침
- "기본 피해 +10%" 처리 → "피해 +10%" 패턴이 또 매칭

**시도 1: Negative lookbehind**
```javascript
(?<!___PROCESSED___)피해 \+(\d+)%
```
- 일부 브라우저에서 호환성 문제

**시도 2: 마커 방식**
```javascript
"기본 피해 +10%___PROCESSED___"
// 나중에 마커 제거
```
- 여전히 중복 발생

**최종 해결: 매칭 수집 후 역순 교체**
```javascript
// 1. 모든 매칭 찾기
const matches = [];
while ((match = regex.exec(text)) !== null) {
  matches.push({ fullMatch, value, index });
}

// 2. 역순으로 교체 (인덱스 유지)
for (let i = matches.length - 1; i >= 0; i--) {
  const m = matches[i];
  const replacement = `${m.fullMatch} <span>(+${bonus})</span>`;
  text = text.substring(0, m.index) + replacement + ...
}
```

**결과:**
```html
기본 피해 +10% (+21.0%), 추가 피해 +15% (+30%)
```
- 각 수치가 정확히 1번만 보너스 표시

### 문제 2: UIController.js 파일 비대화

## 💡 메모 / 인사이트

### 각인 중복의 깊이

처음에는 "왜 같은 능력을 여러 번 각인하지?"라고 생각했지만:

**실제로는:**
- 특화 빌드 가능 ("연속 공격" 7회 각인 → 연속 공격 특화)
- 선택의 재미 (다양화 vs 특화)
- 비용 증가로 밸런스 유지 (x11.5까지 증가)
- 장기 목표 제공 (7성까지 성장)

**교훈:** 단순해 보이는 시스템도 중복을 허용하면 깊이가 생긴다.

### 비용 곡선의 중요성

선형 증가 (20, 40, 60, 80...):
- 예측 가능하지만 지루함
- 후반에도 쉽게 각인 가능

지수 증가 (x1.0, x1.5, x2.2, x3.2, x4.5...):
- 초반: 저렴 (1-2회 각인 쉬움)
- 중반: 적당 (3-4회 각인 고민)
- 후반: 매우 비쌈 (6-7회 각인 도전적)

**교훈:** 지수 곡선은 자연스러운 난이도 곡선을 만든다.

### 모든 노드를 수치 증폭으로

타겟팅 변경, 즉시 처치 같은 "특수 효과"를 제거하고 모두 수치 증폭으로:

**장점:**
- 각인 중복 시 명확한 효과
- 밸런싱이 수치로 가능
- 유저가 성장을 숫자로 확인 가능

**단점:**
- 다양성 감소? (모두 피해 증폭)
- 특수한 느낌 감소?

**실제로는:**
- 조건부 증폭으로 충분히 다양함 (풀피, 저체력, 중장갑, 연속 공격...)
- 수치 증폭도 강력하면 특수하게 느껴짐 (+40% 피해)
- 밸런싱과 안정성이 더 중요

**교훈:** 단순한 시스템이 종종 더 강력하다.

### 정규식의 함정

패턴 중복 문제:
- "기본 피해 +10%" 처리
- "피해 +10%" 패턴이 또 매칭

**해결 시도:**
1. Marker 방식 → 여전히 중복
2. Negative lookbehind → 호환성 문제
3. 매칭 수집 후 역순 교체 → 성공 ✅

**교훈:** 정규식은 강력하지만, 패턴 중복을 항상 조심해야 한다. 때로는 명시적 파싱이 더 안전하다.

### UI 간소화의 가치

처음 구현:
- 스탯 게이지바 (60% 채워진 바)
- 변화량 표시 (15 → 16.1 +7%)
- 각인 모듈 정보 (Damage, Trigger)

사용자 요청: "더 간단하게"

최종:
- 게이지바 제거 → 숫자만
- 변화량 제거 → 최종값만
- 모듈 정보 제거 → 효과만

**결과:**
- 화면이 훨씬 깔끔
- 중요한 정보에 집중
- 공간 절약

**교훈:** 정보가 많다고 좋은 것이 아니다. 필요한 정보만 보여주는 것이 더 좋은 UX다.

- 읽기 어려움
- 유지보수 어려움
- 특정 기능 찾기 어려움

StarUpgradeManager로 분리:
