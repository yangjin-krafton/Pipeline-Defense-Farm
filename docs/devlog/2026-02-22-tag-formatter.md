# 2026-02-22 개발 로그: 태그 포맷터 모듈

## 📋 세션 요약
- 코드 내부에서만 쓰던 영문 태그 식별자(`fat`, `dairy`, `mark`, `expose` 등)를 게임 UI에 표시할 때 한국어 명사로 자동 치환하는 `TagFormatter` 모듈 신설
- 업그레이드 노드 카드, 별 승급 선택 카드, 각인 히스토리 카드 등 설명 텍스트가 렌더링되는 3곳 전부에 일괄 적용

---

## 🗣️ 작업 흐름

### 1. 요청 파악

사용자가 설계 문서(`tower-attack-fun-types-2026.md`)에서 `fat` 단어를 선택한 채로 요청:

> "mark, expose, dairy, fat 와 같은 실제 코드에서 사용하는 태그 호칭을 게임·UI 노출할 때 한국어 명사로 치환될 수 있게 별도 string 포맷 처리 모듈 코드 추가해서, 게임에 모든 설명글에 글 포맷 변환 거쳐서 출력하게 해줘요."

핵심 의도:
- 개발자 코드에서는 `fat`, `dairy`, `expose` 같은 영문 태그 그대로 유지
- 플레이어에게 보이는 텍스트는 `[지방]`, `[유제품]`, `[노출]` 형태의 한국어로 변환
- 단발 함수가 아닌 **모듈** 형태로 분리해 추후 태그 추가 시 한 곳만 수정

### 2. 코드베이스 탐색

Explore 에이전트로 전체 구조 파악:
- 효과 텍스트가 실제로 어떤 형태로 저장되는지 확인
  - 예: `'fat 관통 시 방어 -10%(3초), 관통 피해 +8%'`
  - 예: `'dairy 관통 시 관통 거리 +22%, 탄 크기 +10%'`
- 텍스트가 화면에 표시되는 경로 추적
  1. `UpgradeNode.effect` (원본 데이터 저장)
  2. `UIController._getEnhancedEffectText()` (각인 보너스 계산 후 반환)
  3. `UIController` 각인 히스토리 카드 렌더링
  4. `StarUpgradeManager` 별 승급 카드 렌더링

**결론:** 기존에 텍스트를 변환하는 중앙 처리 모듈이 없었고, 숫자 강조(`_getEnhancedEffectText`)만 UIController에 인라인으로 존재.

### 3. 태그 목록 정리

설계 문서와 실제 코드를 대조해 두 그룹으로 분류:

**음식 태그** (적 유닛 속성):
`carb`, `fat`, `protein`, `dairy`, `spicy`, `soda`, `fermented`, `sugar`, `alcohol`

**시너지/상태이상 태그** (전투 효과):
`expose`, `corrode`, `shock`, `mark`, `clustered`

### 4. TagFormatter 모듈 설계 및 구현

`src/js/utils/TagFormatter.js` 신설:
- `TAG_LABELS` 객체: 태그 → `{ label: 한국어, color: 색상코드 }` 매핑
- `formatTagText(text)` 함수: 영문 태그를 색상 강조된 `[한국어]` 스팬으로 교체
- `getTagLabel(tag)` 함수: 스타일 없이 순수 한국어 레이블만 필요할 때 사용

단어 경계 처리: `(?<![a-zA-Z0-9])fat(?![a-zA-Z0-9])` 패턴 사용 — `fatigue` 같은 단어 내부에서 오매칭되지 않도록.

### 5. 적용 지점 연결

3개 파일, 4곳에 포맷터 삽입:

| 파일 | 위치 | 경우 |
|---|---|---|
| `UIController.js` | `_getEnhancedEffectText` 조기 반환 | 각인 0개인 노드 |
| `UIController.js` | `_getEnhancedEffectText` 최종 반환 | 각인 1개 이상인 노드 |
| `UIController.js` | 각인 히스토리 카드 렌더링 | 획득 이력 표시 |
| `StarUpgradeManager.js` | 별 승급 선택 카드 렌더링 | 각인 선택 패널 |

### 6. 적용 확인

사용자가 "이제 게임에 적용해줘"라고 요청 → 모든 파일이 native ES module 구조라 별도 빌드 없이 저장 즉시 반영됨을 확인하고, 4곳 적용 지점을 재독해 검증.

## 🎯 구현된 기능

### 태그 한국어 변환 시스템

**무엇인가:**
코드 안에 영문으로 된 태그 식별자(`fat`, `dairy`, `expose` 등)가 업그레이드 설명에 그대로 노출되던 문제를 해결. 이제 플레이어에게는 색상이 입혀진 한국어 `[지방]`, `[유제품]`, `[노출]` 형태로 표시된다.

**왜 필요했나:**
- `'fat 관통 시 방어 -10%'` 같은 설명은 개발자에게는 직관적이지만 플레이어에게는 낯선 영문 단어
- 음식 테마 게임이므로 태그명도 맥락에 맞는 한국어로 표시해야 몰입감 유지

**어떻게 동작하나:**
1. 업그레이드 노드 카드, 별 승급 선택 카드, 각인 히스토리 카드가 텍스트를 화면에 그리기 직전에 포맷터를 통과
2. 포맷터가 영문 태그를 찾아 각각의 한국어 레이블과 지정 색상의 `<span>`으로 교체
3. 태그 종류를 추가하거나 수정할 때는 `TagFormatter.js`의 `TAG_LABELS` 한 곳만 수정하면 전체 반영

## 🤔 결정 사항 & 논의

### 데이터 레벨 vs 렌더 레벨 변환

**선택지:**
- A) `UpgradeNode` 생성 시점에 effect 문자열 자체를 한국어로 변환해서 저장
- B) 렌더링 시점에 변환 (저장 데이터는 영문 유지)

**결정:** B) 렌더링 시점 변환

**이유:**
- 원본 영문 태그 이름이 JavaScript 로직(조건 검사, 모듈 처리)에서도 동일하게 쓰임
- 데이터와 표시 문자열을 분리해야 나중에 다국어 지원이나 포맷 변경이 쉬움
- 한 함수만 거치면 되므로 적용 지점 관리가 단순

### 태그 색상 구분

각인 숫자 강조(`#ffd700` 금색)와 겹치지 않도록 태그별 의미에 맞는 색상 배정:
- 음식 태그: 음식 색감 (주황, 녹색, 하늘색, 분홍 등)
- 시너지 태그: 효과 성격 (노출=주황, 부식=연두, 감전=하늘, 표식=자홍, 군집=노랑)

## 💡 메모 / 인사이트

### 중앙 처리 모듈의 가치

기존에는 숫자 강조(`_getEnhancedEffectText`) 하나만 UIController에 인라인으로 있었다. 태그 변환도 거기에 덧붙일 수 있었지만, 별도 모듈(`TagFormatter.js`)로 분리한 이유:

- **단일 책임**: UIController는 UI 조합, TagFormatter는 텍스트 변환만 담당
- **재사용성**: StarUpgradeManager, 미래의 다른 UI 파일에서 같은 함수 import
- **확장성**: 태그 추가·수정·삭제 시 한 파일만 열면 됨

### 렌더 직전 변환의 장점

데이터는 원본 그대로, 표시만 바꾸는 패턴은 다른 곳에도 적용할 여지가 있다:
- 수치 포맷팅 (소수점, 단위 표시)
- 조건부 색상 강조 (임계값 이상이면 빨간색 등)
- 텍스트 축약 (긴 이름을 화면 크기에 맞게)
