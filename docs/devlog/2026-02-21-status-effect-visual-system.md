# 2026-02-21 개발 로그: 상태 이상 시각화 시스템

## 📋 세션 요약
- 음식 enemy의 HP 바가 작동하지 않는 문제 발견 및 해결
- 문서 기준 5가지 상태 이상(expose, corrode, shock, mark, clustered)을 HP 바 색상과 아이콘으로 시각화
- HP 바를 기본 빨간색으로 변경하고, 상태 이상에 따라 자동으로 색상이 변경되는 시스템 구현

---

## 🗣️ 작업 흐름

### 1. HP 바가 보이지 않는 문제 발견

**사용자 보고:**
- HP 바가 머리 위에 있지만 동작하지 않음
- 어두운 빨간색 배경만 보이고 안쪽 게이지(전경)가 전혀 안 보임

**원인 파악:**
LLM이 코드를 탐색하며 문제 분석:
1. HPBarRenderer가 WebGL2 Instanced Rendering으로 구현되어 있음
2. 배경과 전경을 2번의 draw call로 그림
3. 콘솔 로그 확인 결과: HP가 100%로 정상 표시됨
4. **문제 발견**: 전경 렌더링 시 VAO(Vertex Array Object)가 unbind되어 attribute 설정 실패

**첫 번째 시도: 셰이더 수정**
- HP 바 전경이 중앙 정렬에서 왼쪽 정렬로 변경
- HP가 줄어들 때 오른쪽이 줄어들도록 개선
- 하지만 여전히 전경이 보이지 않음

**두 번째 시도: VAO 바인딩 문제 해결**
- 기존 코드: 배경 렌더링 → VAO unbind → 전경 렌더링 시도 → 실패
- 수정: VAO를 한 번만 바인딩하고 두 draw call을 연속 실행
- 결과: **성공!** HP 바 전경이 정상적으로 표시됨

### 2. 상태 이상 시스템 구현

사용자가 문서(`tower-attack-fun-types-2026.md`)를 참고하여 상태 이상 시각화를 요청:
- 음식 enemy에 적용 가능한 상태 이상 효과를 HP 바에 표시
- 문서 기준 5가지 상태 이상 타입 적용 필요

**LLM의 접근:**

**1단계: 기존 구조 탐색**
- Explore 에이전트로 enemy HP 바, 상태 이상 시스템 파일 탐색
- 발견: StatusModule, TagBonusModule, BaseTower 등 이미 구조는 갖춰져 있음
- HPBarRenderer는 WebGL2, 상태 이상은 Canvas 2D로 분리 가능

**2단계: StatusModule 업데이트**
- 기존 타입: `slow`, `armorReduction`, `stun`, `acid`, `mark`
- 문서 기준으로 확장:
  - `expose` (점막 취약 노출): 소화 피해 +8%
  - `corrode` (산성 부식): 소화 저항 -6%
  - `shock` (연동 교란): 이동 속도 -10%
  - `mark` (분해 표식): 치명 보정 +8%p
  - `clustered` (정체 군집): 군집 범위 +10%

**3단계: StatusEffectRenderer 생성**
- HP 바 위에 작은 아이콘으로 상태 이상 표시
- 각 상태마다 고유 색상 + 이모지:
  - 🎯 expose (주황색)
  - 🧪 corrode (초록색)
  - ⚡ shock (노란색)
  - 🔴 mark (빨간색)
  - 👥 clustered (보라색)
- 중복 제거: 같은 타입은 하나만 표시
- 만료 확인: 지속 시간 지난 효과는 자동 숨김

**4단계: GameLoop 통합**
- 렌더링 파이프라인에 상태 이상 표시 추가
- Canvas 2D의 clear() 문제 해결: drawEmojis() 내부에서 상태 이상 렌더링
- 렌더링 순서: WebGL(HP바) → Canvas(상태 아이콘) → Canvas(이모지)

**5단계: TagBonusModule 매핑**
- 기존 `armorReduction` → `corrode`로 자동 매핑
- 기존 `slow` → `shock`로 자동 매핑
- 5가지 새 타입 직접 지원 추가

### 3. HP 바 색상 시스템 개선

사용자가 추가 요청:
- "HP 바 기본 색은 빨간색만 사용"
- "나중에 상태 이상 표시를 위해 색상 변경 남겨놔요"

**구현:**

**기본 색상:**
- 배경: 어두운 빨강 (항상 고정)
- 전경: 빨간색 (기본 상태)

**상태 이상별 HP 바 색상 자동 변경 (우선순위 순):**
1. corrode (산성 부식) → 초록색
2. shock (연동 교란) → 노란색
3. expose (점막 취약) → 주황색
4. mark (분해 표식) → 보라색
5. clustered (정체 군집) → 하늘색
6. stun (기절) → 회색

**구조:**
- `_getBarColor(food, hpPercent)` 메서드로 색상 결정
- 활성 상태 이상 필터링 후 우선순위에 따라 색상 반환
- 확장 가능: 새 상태 추가 시 switch 문에 case만 추가하면 됨

**디버그 로그 제거:**
- 초기 문제 해결용 로그 제거 (1초마다 HP 정보 출력하던 부분)

## 🎯 구현된 기능

### 1. HP 바 시스템 수정
- **문제**: HP 바 전경이 렌더링되지 않음
- **해결**: WebGL VAO 바인딩 순서 수정
- **개선**: 왼쪽 정렬 (HP가 줄면 오른쪽이 줄어듦)

### 2. 상태 이상 아이콘 표시
- HP 바 위 8픽셀에 작은 원형 아이콘 표시
- 각 상태마다 고유 색상 배경 + 이모지
- 여러 상태가 동시에 있으면 가로로 나열 (10픽셀 간격)
- 지속 시간 끝나면 자동으로 사라짐

### 3. HP 바 색상 자동 변경
- 기본 상태: 빨간색
- 상태 이상 적용 시: 자동으로 색상 변경
- 우선순위 시스템: 여러 상태가 있어도 가장 중요한 것만 HP 바에 반영
- 예시:
  - 산성 부식(corrode) → HP 바가 초록색으로
  - 감전(shock) → HP 바가 노란색으로
  - 취약(expose) → HP 바가 주황색으로

### 4. 문서 기준 5가지 상태 이상 통합
- 문서(`tower-attack-fun-types-2026.md`)의 시너지 태그 시스템 반영
- StatusModule에 5가지 타입 추가
- TagBonusModule에서 자동 매핑 (하위 호환)
- 타워 노드에서 쉽게 사용 가능

## 🤔 결정 사항 & 논의

### HP 바와 아이콘의 이중 표시
- **선택**: HP 바 색상 + 아이콘 둘 다 표시
- **이유**:
  - HP 바 색상: 멀리서도 한눈에 파악 가능
  - 아이콘: 여러 상태를 동시에 확인 가능
  - 우선순위: HP 바는 가장 중요한 상태 하나만, 아이콘은 모두 표시

### 렌더링 레이어 분리
- **선택**: HP 바는 WebGL, 아이콘은 Canvas 2D
- **이유**:
  - HP 바: 200개까지 instanced rendering으로 고성능
  - 아이콘: 복잡한 이모지와 원형 그리기는 Canvas가 편리
  - 성능: 각 레이어의 장점을 살림

### 기본 HP 바 색상을 빨간색으로
- **사용자 요청**: "기본 색은 빨간색만"
- **이전**: HP %에 따라 초록→노랑→빨강
- **변경 후**: 항상 빨간색 (상태 이상만 색 변경)
- **이유**: 상태 이상 표시와 혼동 방지

## 🐛 문제 & 해결

### 문제 1: HP 바 전경이 안 보임
**증상**: 어두운 빨강 배경만 보이고 게이지가 없음

**원인**: WebGL VAO unbind 타이밍 문제
- 배경 렌더링 후 `render()` 메서드가 VAO를 unbind
- 전경 렌더링 시 VAO 없이 attribute 설정 시도 → 실패

**해결**:
1. 모든 데이터를 GPU로 먼저 전송
2. VAO를 한 번만 바인딩
3. 배경과 전경을 연속으로 drawArraysInstanced 호출
4. 마지막에 한 번만 unbind

**결과**: HP 바가 정상적으로 표시됨

### 문제 2: 상태 이상 아이콘이 지워짐
**증상**: 아이콘을 그려도 이모지 렌더링 시 사라짐

**원인**: Canvas 2D의 `clear()` 호출 순서
- drawStatusEffects() → 아이콘 그림
- drawEmojis() → clear() → 아이콘 사라짐

**해결**: drawEmojis() 내부에서 상태 이상도 함께 렌더링
- clear() 호출 직후 상태 이상 아이콘 렌더링
- 그 다음 이모지 렌더링
- 렌더링 순서 보장

- WebGL2 Instanced Rendering: 대량의 단순한 도형(HP 바)에 최적

- 두 레이어를 겹쳐서 각각의 장점을 살릴 수 있음

- 단계적으로 범위를 좁혀나가는 과정이 핵심
