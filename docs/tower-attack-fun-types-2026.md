# 2026 기준 타워디펜스 타워 설계 마스터 문서 (큰그림 -> 세부그림)

## 문서 목적

- 전 타워를 같은 기준으로 설계해, 업그레이드 시 컨셉 중복/충돌을 방지
- `모듈 재사용`으로 구현 비용을 줄이고 밸런스 조정 단위를 통일
- 먼저 `큰 게임플레이 분기`를 확정하고, 이후 개별 타워 세부 노드를 설계

## 0) 공통 설계 원칙 (충돌 방지)

- 한 타워는 1개의 `주 역할` + 1개의 `보조 역할`만 가진다.
- 다른 분류의 시그니처를 빼앗지 않는다.
  - 예: 단일 화력은 광역 섬멸 성능을 가져가면 안 됨.
  - 예: 군중 제어는 단독 고위협 적 삭제 성능을 가져가면 안 됨.
- 업그레이드 포인트는 기본(0성) `5포인트`, 1성부터 `6포인트` 시작, 최대 `12포인트(7성)`까지 확장한다. 노드는 `10~15개`.
- 노드 위치는 공통으로 `분기(1~3) / 중간(4~9) / 끝(10~12)`를 사용.
- 각 구간마다 강선택 2개 이상을 제공한다.

## 1) 모듈 재사용 사전 (전 타워 공통)

|---|---|---|---|

모듈 사용 권장:
- `타워 전투 노드`: `TM/PM/DM/TB/SM/TR/SF/AM/UM`
- `성장/재화/승급 시스템`: `XM/CM/BM/EM/GM/IM`

### 1-A) 음식(Enemy) 속성 스키마 (모듈 통합)

#### 필수 속성 (모든 음식 공통)

| 필드 | 타입 | 설명 | 참조 모듈 |
|---|---|---|---|
| `id` | string | 음식 고유 식별자 | 전체 |
| `name` | string | 표시 이름 | UI/로그 |
| `tags` | string[] | 음식 태그 목록 (`carb`, `fat`, `protein`, `dairy`, `spicy`, `soda`, `fermented`) | `TB`, `TM`, `SM` |
| `hp` | number | 기본 체력(소화 난이도) | `DM`, `TR` |
| `speed` | number | 이동 속도 | `SM`, `TM` |
| `armor` | number | 물리/일반 피해 감소치 | `DM`, `SM` |
| `resist` | object | 상태저항 (`acid`, `slow`, `stun`, `mark`) | `SM`, `TB` |
| `size` | string | 충돌 크기 (`small`, `medium`, `large`) | `PM`, `TM` |
| `threat` | number | 위협도(우선 타겟팅 용도) | `TM` |
| `traits` | string[] | 특수 특성 (`heavy`, `split`, `shielded`, `swift`) | `TM`, `TR`, `DM` |

#### 권장 속성 (상성/리스크 정밀화)

| 필드 | 타입 | 설명 | 참조 모듈 |
|---|---|---|---|
| `digestStageBias` | object | 기관별 취약/강점 (`stomach`, `smallIntestine`, `largeIntestine`) | `TB`, `TR` |
| `troubleOnLeak` | object | 누수 시 트러블 기여도 (`gas`, `inflammation`, `constipation`) | `RM`, 시스템 |
| `acidLoad` | number | 산성 부담치(과산 시스템 연동) | `SF`, `SM` |
| `greaseLoad` | number | 지방 정체 유발치 | `SM`, `TB` |
| `fermentValue` | number | 발효/가스 누적치 | `SM`, `TR` |
| `nutritionValue` | number | 처치 보상 기본 영양 | `RM` |

#### 모듈별 최소 참조 계약

| 모듈 | 반드시 읽는 음식 필드 | 처리 목적 |
|---|---|---|
| `TM` | `threat`, `traits`, `speed`, `tags` | 우선순위 타겟 선택 |
| `PM` | `size`, `speed` | 탄속/관통/명중 계산 |
| `DM` | `hp`, `armor`, `traits` | 최종 피해 계산 |
| `TB` | `tags`, `digestStageBias` | 음식 상성 보정 |
| `SM` | `resist`, `tags`, `greaseLoad`, `fermentValue` | 상태 부여/저항 판정 |
| `TR` | `hp`, `traits`, `tags` | 조건 발동(임계치/처치시) |
| `RM` | `nutritionValue`, `traits` | 영양/에너지 환급 |
| `SF` | `acidLoad`, `traits` | 과열/과산 리스크 완화 연산 |

#### 음식 데이터 예시 (JSON)

```json
{
  "id": "food_cheese_burger_combo",
  "name": "치즈버거 세트",
  "tags": ["fat", "protein", "dairy", "soda"],
  "hp": 420,
  "speed": 0.82,
  "armor": 18,
  "resist": { "acid": 0.2, "slow": 0.1, "stun": 0.25, "mark": 0.0 },
  "size": "large",
  "threat": 88,
  "traits": ["elite"],
  "digestStageBias": { "stomach": -0.05, "smallIntestine": 0.1, "largeIntestine": 0.2 },
  "troubleOnLeak": { "gas": 4, "inflammation": 3, "constipation": 2 },
  "acidLoad": 3,
  "greaseLoad": 5,
  "fermentValue": 2,
  "nutritionValue": 7

```

#### 검증 규칙 (데이터 입력 시)

- `tags`는 최소 1개 이상, 미등록 태그 금지
- `hp`, `speed`, `nutritionValue`는 0 초과
- `resist` 값은 `0.0 ~ 0.9` 범위
- `heavy` trait이면 `threat >= 15` 권장
- `soda` 태그가 있으면 `fermentValue` 또는 `troubleOnLeak.gas`가 1 이상

#### 테마별 대표 Enemy 강도 프리셋 (config.js 기준 검토안)

검토 요약 (`src/js/config.js`):
- 현재 음식 스탯 범위는 대략 `hp 88~182`, `armor 0~14`, `speed 82~122`, `threat 7~18`, `digestionNeed 66~132`
- 캐주얼 진행에 맞게 `일반/강화/고위협` 3단계로 시작하는 구성이 안전

강도 밴드(권장):
- `Normal`: `threat 7~11`, `digestionNeed 70~98`
- `Strong`: `threat 12~15`, `digestionNeed 99~120`
- `Elite`: `threat 16~18`, `digestionNeed 121~135`

- 이모지 중복 금지: 한 이모지는 하나의 대표 Enemy만 담당

## 2) 전 타워 타입 큰 게임플레이 분기 표 (확장본)

> 이 표를 먼저 고정하고, 세부 노드 설계는 이 표의 분기 안에서만 진행.

| 분류 | 타워 타입 | 큰 게임플레이 분기 (상위 3축) | 고유 역할(침범 금지선) | 주 송신 태그(5종) | 주 수신 태그(5종) | 우선 모듈 조합 |
|---|---|---|---|---|---|---|
| 단일 화력 | 스나이퍼/레일건 | `암살`/`약점분해`/`임계마무리` | 고위협 1체 정리 | `mark` | `expose`, `shock` | `TM+DM+TR` |
| 단일 화력 | 차지 캐논 | `완충폭딜`/`유연운영`/`연계폭발` | 타이밍 기반 단일 고점 | `mark` | `expose`, `shock` | `PM+DM+SF+TR` |

| 특수 메커닉 | 시간 왜곡 | `둔화장`/`가속장`/`쿨조작` | 전장 템포 제어 | `shock`, `clustered` | `mark`, `corrode` | `SM+AM+TR` |
| 특수 메커닉 | 임계 마무리포 | `임계마무리`/`표식마무리`/`연계마무리` | 마무리 전담 | `mark` | `expose`, `shock` | `TR+DM+TM` |

### 2-A) 타워 간 연동/증폭 공통 레이어 (버프 재사용)

검토 결론:
- 전 타워에 개별 전용 버프를 만들기보다, `공통 시너지 태그`를 재사용하는 쪽이 충돌이 적고 유지보수에 유리
- 버프는 `송신(Source)`과 `수신(Consumer)`을 분리하고, 수신 측에서 동일 규칙으로 처리
- 공통 레이어는 복잡도 관리를 위해 `5종`으로 제한

| 시너지 태그 | 소화 테마 이름 | 송신(주로 생성) | 수신(주로 소비) | 공통 효과(권장) | 상한/안전장치 |
|---|---|---|---|---|---|
| `expose` | 점막 취약 노출 | 취약 표식, 방어 파쇄 | 단일 화력, 관통 볼트 | 소화 피해 +8% | 최대 3중첩, 총 +24% |
| `corrode` | 산성 부식 상태 | 독/산성, 방어 파쇄 | 지속 피해, 범위 화력 | 소화 저항 -6% | 최대 2중첩 |
| `shock` | 연동 교란 자극 | 연쇄 번개, EMP | 범위 화력, 임계 마무리포 | 장운동 속도 -10%(이동 둔화) | 감속 합산 캡 -35% |
| `mark` | 분해 표식 | 스나이퍼, 취약 표식 | 마무리/차지 캐논 | 치명 보정 +8%p | 고위협 대상 +5%p로 제한 |
| `clustered` | 정체 군집 상태 | 넉백/흡인, 함정 바닥 | 박격포, 산탄, 자폭 소환 | 군집 대상 범위 +10% | 반경 합산 캡 +25% |

### 2-B) 분류별 연동 설계 가이드

| 분류 | 주 송신 태그 | 주 수신 태그 | 연동 의도 |
|---|---|---|---|
| 단일 화력 | `mark` | `expose`, `shock` | 마무리 정확도 강화 |
| 범위 화력 | `shock`, `clustered` | `corrode`, `clustered` | 군집 붕괴 속도 상승 |
| 지속 피해 | `corrode` | `clustered`, `mark` | 누적 피해 효율 극대화 |
| 군중 제어 | `clustered`, `shock` | `mark` | 전장 정렬로 팀 딜 지원 |

| 특수 메커닉 | `shock`, `mark` | `mark`, `expose` | 고숙련 콤보 완성 |

### 2-C) 버프 재사용 구현 규칙

- 모든 버프/디버프는 문자열 ID(`expose`, `corrode`, `shock`, `mark`, `clustered`)로 통일
- 적용 순서: `TagBonus(TB) -> Status(SM) -> Trigger(TR) -> Damage(DM)`
- 동일 ID 중첩은 `스택 증가`만 허용, 같은 프레임 재적용은 갱신만 수행
- `## 7` 연동: 2x/3x 배속 시 시너지 발동 횟수는 시간 보정 후 캡 적용

## 4) 세부 타워 설계 (2026 기준)

### 공통 규칙

- 포인트: 기본(0성) `5`, 1성 `6` (최대 7성 `12`)
- 노드: `12`
- 위치: `분기 1~3 / 중간 4~9 / 끝 10~12`
- 시작 분기 규칙: 각 타워는 `분기 1~3` 중 `2~3개`를 시작 시 즉시 선택 가능(고정 스타트 금지)
- 중간 노드 선행 규칙: `4~7`은 선택한 시작 분기(`1~3`)와 교차 연결, `8~9`는 복수 중간 노드 경유 허용으로 빌드 전환 여지 확보
- 위치별 강선택:
  - 분기: 즉시 고점 vs 운영 안정
  - 중간: 상성 대응 vs 리스크 완화
  - 끝: 피니시 폭딜 vs 경제/안정 마감
- `## 7` 연동 밸런스:
  - 과도한 원턴 마무리/연쇄 폭딜 계수 하향(캐주얼 피로 완화)
  - 배속 사용 시 끝 노드 계열 발동 효율 캡 적용: `2배속 95%`, `3배속 85%`
- `## 2` 연동 태그 규칙:
  - 세부 노드의 연동/증폭은 `expose`, `corrode`, `shock`, `mark`, `clustered`만 사용
  - 신규 노드 추가 시 5개 태그 외 신규 태그 금지

### 7-5) 진화 규칙 (1성 -> 7성)

- 최대 레벨/트리 포인트: `Lmax = 5 + 성급`
- 1성 `Lv6/포인트6` -> 7성 `Lv12/포인트12`
- 승급 시 랜덤 스탯 증가 적용

| 스탯 | 기본 랜덤 증가 | 하한 보호 |
|---|---|---|
| 공격력 | `+5% ~ +11%` | `+7%` 미만 시 `+7%`로 보정 |
| 공격속도 | `+1% ~ +4%` | `+2%` 미만 시 `+2%`로 보정 |
| 사거리 | `+0% ~ +3%` | `+1%` 미만 시 `+1%`로 보정 |
| 상태이상 성공률 | `+1%p ~ +3%p` | `+1.5%p` 미만 시 `+1.5%p` |

**성급별 XP 배율 (지수 곡선):**
- 공식: `multiplier = 1.0 + (성급 - 1)^1.4 * 0.5`
- 1성: x1.0, 2성: x1.5, 3성: x2.5, 4성: x4.0, 5성: x6.0
- 6성: x8.5, 7성: x11.5

추가 구제:

### 7-7A) 전투 배속 기능 재화화 (2x / 3x)

적용 제안:
- 기본 배속(1x)은 무료
- 2배속/3배속은 `SC` 소모형 부스터로 전환
- 배속은 전투 단위 버프(중첩 불가), 종료 시 자동 해제

| 배속 모드 | 지속 시간 | SC 비용 | 설계 의도 |
|---|---|---|---|
| 2배속 | 10분 | 6 | 캐주얼 반복 플레이의 기본 가속 |
| 3배속 | 10분 | 12 | 빠른 소모용 프리미엄 가속 |

운영 규칙(권장):
- 웨이브 시작 후 30초 이내에는 무료 취소 가능(오입력 방지)
- 동일 전투에서 3배속은 최대 2회까지만 사용 가능(과소모 방지)
- 자동 전투/완전 방치 중에는 3배속 효율 80%로 제한(접속 플레이 보상 유지)

### 7-8) SC 소비 우선순위 시뮬레이션 (시간 접속 루프)

성장 가속 효과 고정:
- `12 SC` 소비 -> `1시간 동안 XP +40%` (중첩 불가) ✅ **구현 완료**
- `8 SC` 소비 -> `30분 동안 NC +25%` (중첩 불가) ⏳ **구현 예정**

| 사용처 | 비용 | 기대 효율(권장 해석) | 구현 상태 |
|---|---|---|---|
| 성장 가속 1회 | 12 SC | XP 추가 40% (단기 성장) | ✅ 완료 |
| 시간 부스터 1회 | 8 SC | NC 수급 +25% (30분) | ⏳ 예정 |
| 승급 리롤 1회 | 20 SC | 단일 스탯 품질 개선 | ✅ 완료 |
| 각인 후보 확장 | 30 SC | 장기 파워 고점 선택권 확보 | ⏳ 예정 |

권장 가이드:
- 초반(1~4성): 성장 가속 위주 사용 (빠른 성장 우선)
- 중반(5~6성): 가속/리롤 혼합 (품질 개선 시작)
- 후반(7성): 각인 후보 확장 우선 (장기 파워 확보)

**시간 접속 유도 루프 (1시간 단위):**
1. 접속 시 `+10 SC` 시간 보급 수령
2. 성장 가속(`12 SC`) 즉시 소비 → 1시간 플레이
3. 남은 SC(-2)는 시간 경과로 회복 (시간당 +20 SC)
4. 다음 1시간 후: SC 회복 완료 → 반복

**6시간 접속 루프 (아침/점심/저녁/새벽):**
1. 6시간마다 대량 보상: `+500 NC`, `+20 SC` ⏳ **구현 예정**
2. 누적 SC로 승급 리롤 또는 각인 확장
3. 누적 NC로 신규 타워 설치 또는 스킬트리 확장

### 7-9) 영구 스킬트리 각인 규칙 (계산식 고정)

- 만랩 승급 시 각인 1개 획득
- 기본: 후보 1개 랜덤
- SC 30 사용: 후보 3개 중 1개 선택
- 각인 리롤 불가
- 다음 승급에서도 추가 각인 가능, 동일 계열 중첩 허용

중첩 계산식:
- `최종 계수 = Base * (1 + 트리합 + 각인합)`
- 동일 계열 각인 중첩 캡: `3`
- `트리합 + 각인합` 합산 캡: `+120%`

### 7-10) 운영 메모 (2026-02-21 개정)

**핵심 플레이 루프:**
1. **첫 10분 (온보딩)**: 타워 Lv5 만랩 체감 → 승급 경험 → 스킬트리 활성화
2. **첫 30분 (코어 루프)**: 2-3개 타워 운영, 2-3성 도달, NC 획득 체감
3. **1시간 루프 (지속 성장)**: SC +10 수령 → 성장 가속 소비 → 1시간 플레이
4. **6시간 루프 (대량 보상)**: 아침/점심/저녁/새벽 접속 → NC/SC 대량 보상

**재화 역할 분리:**
- **NC (누적형)**: 장기 누적, 타워 설치/노드 활성화/승급 기본비
- **SC (회복형)**: 일일 루프/선택 압박, 성장 가속/리롤/각인 확장

**접속 유도 설계:**
- SC 상한 80으로 자주 소비 유도
- 1시간마다 +10 SC 수령 (수동)
- 6시간마다 대량 보상 (아침 6시, 점심 12시, 저녁 6시, 새벽 12시)
- 오버플로 50% NC 변환으로 방치 손실 완화

**초반 보호:**
- 1~2성 첫 트리 리셋 무료 1회 유지
- 첫 10분 빠른 만랩으로 즉시 재미 제공
- 2초 틱으로 경험치 상승 체감 강화
