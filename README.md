# Pipeline Defense Farm

브라우저에서 실행하는 소화기관 테마 타워 디펜스 프로토타입입니다. 현재 구현은 초기 기획서에 적혀 있던 "소화 효율 시뮬레이션" 전체를 완성한 상태라기보다, `명절 과식 디펜스`라는 이름의 싱글 화면 플레이어블 버티컬 슬라이스에 가깝습니다.

음식 이모지 적이 `밥위 / 디저트위 / 술위` 입구에서 들어와 `소장 / 대장` 경로를 따라 이동하고, 플레이어는 빈 슬롯에 타워를 설치해 막습니다. 전투 자체보다도 "타워를 깔고, 성장시키고, 승급시키고, 다시 저장해서 이어서 플레이하는 루프"가 현재 프로토타입의 중심입니다.

## 이 프로토타입이 실제로 구현한 것

### 1. 현재 플레이 루프

- 시작 오버레이에서 미니게임과 3페이지 튜토리얼을 본 뒤 본게임에 진입합니다.
- 맵은 소화기관 모티프의 고정 경로 5개로 구성되어 있습니다.
- 음식 적은 시간표 웨이브가 아니라 `압박도 기반 연속 스폰` 방식으로 계속 등장합니다.
- 플레이어는 타워 슬롯을 탭해 타워를 설치하거나, 이미 설치한 타워의 상세 패널을 열 수 있습니다.
- 하단 시트에서 타워 정보, 스킬 트리, 승급 UI, 잠금 슬롯 해제를 처리합니다.
- 게임 상태는 `localStorage`에 자동 저장되고, 재접속 시 오프라인 보상이 계산됩니다.

### 2. 현재 들어간 전투/성장 시스템

- 타워 3종이 구현되어 있습니다.
  - `위산 레일 주입기`: 보스/엘리트 암살형
  - `효소 축전 캐논`: 충전형 단일 화력
  - `연동 관통 볼트기`: 직선 관통형
- 적 데이터는 밥위, 디저트위, 술위 테마로 나뉜 음식 이모지 테이블을 사용합니다.
- 타워는 2초마다 XP를 얻고 레벨업하며, 레벨업 시 업그레이드 포인트를 받습니다.
- 스킬 트리 노드를 찍어 성능을 강화할 수 있습니다.
- 만렙 + 포인트 소진 조건을 만족하면 성급 승급 UI가 열립니다.
- 승급 시 랜덤 스탯 롤, SC 리롤, 각인 선택, 영구 각인 저장이 동작합니다.
- NC/SC 2종 재화, 시간당 SC 회복, 1시간 보상, 6시간 보상이 구현되어 있습니다.

### 3. 지금 보이는 제품 형태

- `WebGL2 + Canvas` 기반 렌더링
- 세로형 모바일 HUD를 전제로 한 단일 화면 UI
- BGM, 효과음, 오디오 설정 패널
- 저장/로드, 오프라인 보상 모달
- 경로 편집기, 타워 슬롯 편집기, 개발자 콘솔 명령어

즉, 이 저장소는 더 이상 "경로만 흐르는 데모"가 아니라, 전투와 메타 성장까지 연결된 브라우저 게임 프로토타입입니다.

## AI 사용 여부

현재 코드베이스에는 런타임 AI 기능이 없습니다.

- `OpenAI`, `LLM`, `agent`, `prompt`, `memory` 같은 호출 경로가 없습니다.
- 게임 플레이 중 모델이 의사결정을 하거나 콘텐츠를 생성하지 않습니다.
- 서버 연동형 AI 인프라도 없습니다.

대신 오프라인 제작용 유틸리티는 있습니다.

- `src/utils/chiptune_generator.py`
- `src/utils/sfx_16bit_generator.py`
- `src/utils/upgrade_sfx_generator.py`

이 스크립트들은 오디오 에셋 제작용 도구이며, 현재 저장소에서 확인되는 범위에서는 AI 기능이 아니라 로컬 생성 유틸리티에 가깝습니다.

## 무엇을 시도했고, 결과가 어땠는가

이 프로젝트가 시도한 핵심은 "소화기관 테마를 가진 독특한 타워 디펜스"를 실제로 손에 잡히는 프로토타입으로 압축하는 일이었습니다.

구현 결과를 기준으로 보면 다음이 작동합니다.

- 소화기관 테마를 시각적으로 읽히는 경로 구조로 바꾸는 것
- 음식 이모지 적과 태그 기반 상성을 전투 데이터로 옮기는 것
- 단순 설치형 TD를 넘어서 타워 성장, 승급, 각인까지 연결하는 것
- 로컬 저장과 오프라인 보상으로 세션형 플레이를 이어가는 것
- 모바일 감각의 하단 시트 UI와 튜토리얼을 붙여 처음 진입한 사용자도 흐름을 이해하게 만드는 것

반대로, 초기 큰 그림에서 더 좁혀진 부분도 분명합니다.

- 현재 전투는 "소화 효율 퍼즐"보다 "공격형 타워 디펜스" 쪽에 더 가깝습니다.
- 지원형/기관 전용 타워의 폭넓은 조합은 아직 없습니다.
- 식사 기록을 실제 웨이브로 바꾸는 체크인 시스템은 구현되지 않았습니다.
- 오프라인 보상은 수치 계산 모달로 단순화되었습니다.

## 초기 계획과 현재 구현의 차이

| 초기 구상 | 현재 코드 기준 구현 | 독자에게 중요한 의미 |
| --- | --- | --- |
| 소화기관 전체를 활용한 기관별 규칙 중심 TD | 5개 경로를 가진 단일 맵 위에서 싸우는 브라우저 전투 프로토타입 | 테마는 유지됐지만, 시스템 중심보다는 플레이 가능한 버티컬 슬라이스에 집중되어 있습니다. |
| 식사 체크인으로 현실 음식이 오늘의 웨이브가 되는 방치 루프 | 저장 시간 기준 XP/NC/SC를 주는 오프라인 보상 모달 | 현재 방치 설계는 더 단순하고 즉시 검증 가능한 형태입니다. |
| 효소/산/담즙/프로바이오틱스 등 폭넓은 역할군 | 실제 구현 타워는 3종 공격형 타워 중심 | 밸런스와 조합보다 기본 전투 감각과 성장 UI 검증이 우선이었습니다. |
| localStorage + IndexedDB 스냅샷 | `SaveSystem` 기준 `localStorage` 저장 | 실행 장벽은 낮지만 저장 구조는 로컬 단일 브라우저에 묶입니다. |
| 큰 설계 문서 중심 진행 | 실제 코드에는 승급, 각인, UI 리팩터링, 개발자 툴까지 들어감 | README를 기획서가 아니라 "지금 무엇이 돌아가는지" 중심으로 읽어야 합니다. |

## 현재 범위

### 구현됨

- 브라우저 실행형 정적 게임
- WebGL2 렌더링과 음식 이모지 적
- 타워 설치/판매/잠금 슬롯 해제
- 타워 XP, 레벨, 업그레이드 포인트
- 스킬 트리와 영구 각인 승급 시스템
- NC/SC 재화 루프
- 자동 저장, 저장 복원, 오프라인 보상
- 시작 오버레이, 튜토리얼, 미니게임
- 개발용 에디터와 콘솔 명령어

### 아직 비어 있거나 단순화됨

- AI 플레이어 또는 AI 제작 파이프라인
- 서버 저장, 계정 연동, 멀티플레이
- 식사 체크인 기반 웨이브 생성
- 기관별 트러블 시스템의 완전한 게임화
- 더 넓은 타워 역할군과 장기 밸런싱
- 자동 테스트나 빌드 파이프라인

## 실행 방법

이 저장소에는 `package.json`이 없습니다. 현재 형태는 정적 파일을 로컬 서버로 열어 실행하는 방식이 맞습니다.

```bash
python3 -m http.server 4173 --directory src
```

그 다음 브라우저에서 `http://127.0.0.1:4173` 를 열면 됩니다.

### 실행 환경 메모

- `WebGL2`가 필요합니다.
- 오디오 재생은 첫 사용자 입력 이후 활성화될 수 있습니다.
- 저장은 브라우저 `localStorage`를 사용합니다.

## 개발용 기능

### 내장 편의 기능

- 개발자 메뉴에서 경로 편집기와 타워 슬롯 편집기를 열 수 있습니다.
- `Ctrl+Shift+E`로 경로 편집기를 토글할 수 있습니다.
- 콘솔에서 다음 명령어를 사용할 수 있습니다.
  - `addXP(amount)`
  - `maxLevel()`
  - `towerInfo()`
  - `addNC(amount)`
  - `addSC(amount)`
  - `currency()`
  - `maxCurrency()`

### 폴더 가이드

- `src/index.html`: 전체 화면 구조와 HUD
- `src/js/main.js`: 부트스트랩, 오디오, 저장 복원, 시작 오버레이
- `src/js/game/`: 게임 루프, 스포너, 난이도
- `src/js/digestion/`: 타워/성장/경제/저장 핵심 로직
- `src/js/ui/`: 하단 시트, 승급 UI, 보상 버튼, 각종 패널
- `src/js/renderer/`: WebGL/Canvas 렌더러
- `docs/devlog/`: 최근 작업 기록
- `sandbox/`: 별도 실험용 HTML

## 한계와 주의점

- 현재 README는 기획서가 아니라 "2026-03-12 기준 저장소에 실제로 들어 있는 구현"을 설명합니다.
- 문서에 남아 있는 큰 설계 아이디어 중 일부는 아직 코드에 없습니다.
- 브라우저 상호작용 전체는 이 환경에서 직접 플레이 검증하지 못했습니다.
- 대신 소스 코드를 다시 읽었고, `python3 -m http.server 4173 --directory src`로 정적 서빙이 정상 응답하는 것까지는 확인했습니다.

## 한 줄 요약

`Pipeline Defense Farm`은 지금 기준으로 보면, 소화기관 콘셉트를 가진 대형 시스템 설계 문서가 아니라, 3종 타워와 성장/승급/저장 루프를 갖춘 브라우저용 타워 디펜스 프로토타입입니다.
