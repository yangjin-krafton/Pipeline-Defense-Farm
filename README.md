# Pipeline Defense Farm (소화기관 디펜스)

> 밥 위/디저트 위/술 위 → 소장 → 대장으로 이어지는 "소화 파이프라인" 맵에서
> 다양한 음식(Enemy)이 통과하기 전에, 소화를 돕는 컨셉의 타워를 세워 **분해/중화/흡수**를 최적화하는 타워 디펜스.

---

## 프로젝트 구조

```
src/
├── index.html              # 메인 HTML (간소화)
├── styles/
│   └── main.css           # 스타일시트
├── assets/
│   └── bgm/               # 배경 음악
│       ├── game_theme.wav          # 메인 BGM (66초, 16-bit 칩튠)
│       ├── game_theme_structure.json
│       └── instruments/            # 악기별 WAV (18개)
├── utils/
│   └── chiptune_generator.py  # BGM 생성 스크립트
└── js/
    ├── main.js            # 메인 엔트리 포인트
    ├── config.js          # 게임 설정 상수
    ├── renderer/          # 렌더링 모듈
    │   ├── WebGLRenderer.js    # WebGL2 렌더러
    │   └── EmojiRenderer.js    # Emoji 캔버스 렌더러
    ├── game/              # 게임 로직
    │   ├── GameLoop.js         # 메인 게임 루프
    │   └── FoodSpawner.js      # 음식 스포너
    ├── systems/           # 게임 시스템
    │   ├── FlowSystem.js       # Flow 애니메이션 시스템
    │   └── AudioSystem.js      # BGM/효과음 시스템
    └── utils/             # 유틸리티
        ├── PathFollowerSystem.js  # 경로 추적 시스템
        └── geometry.js            # 지오메트리 함수
```

### 모듈 설명

**렌더링 (renderer/)**
- `WebGLRenderer.js`: WebGL2 기반 고성능 렌더링 (배경, 경로, 애니메이션)
- `EmojiRenderer.js`: Canvas 2D 기반 emoji 렌더링 (캐싱 최적화)

**게임 로직 (game/)**
- `GameLoop.js`: 메인 게임 루프, 업데이트/렌더링 관리
- `FoodSpawner.js`: 음식 생성 및 스폰 타이밍 관리

**시스템 (systems/)**
- `FlowSystem.js`: 경로를 따라 흐르는 애니메이션 시스템
- `AudioSystem.js`: Web Audio API 기반 BGM/효과음 재생

**유틸리티 (utils/)**
- `PathFollowerSystem.js`: 재사용 가능한 경로 추적 모듈
- `geometry.js`: 메시 생성 헬퍼 함수

**음악 생성 (src/utils/)**
- `chiptune_generator.py`: 16-bit 칩튠 BGM 생성기 (Python)

## 현재 데모: Digestive Run

현재 구현된 데모는 "Digestive Run"으로, 소화 경로를 따라 음식(emoji)이 이동하는 시뮬레이션입니다.

### 실행 방법

```bash
# src/index.html을 브라우저에서 열기
# 또는 로컬 서버 실행
npx serve src
```

### 특징

- **WebGL2 최적화**: 고성능 렌더링
- **Emoji 캐싱**: Canvas 2D로 emoji 렌더링 최적화
- **FPS 모니터**: 실시간 성능 측정
- **16-Bit 칩튠 BGM**: 소화 과정에 맞춘 동적 음악 (Web Audio API)
- **모듈화 구조**: 유지보수와 확장이 쉬운 구조

## 게임 컨셉

이 게임의 맵은 **소화 기관 자체**입니다.
음식이 통과하는 경로(밥 위/디저트 위/술 위→소장→대장)는 고정된 "소화 파이프라인"이고, 플레이어는 중간중간 **소화에 도움을 주는 장치(타워)**를 배치해 음식이 문제를 일으키기 전에 처리합니다.

기본 TD와의 차이:

- "끝까지 못 가게 막는 것"보다 **원활히 통과시키는 것(소화 효율)**이 목표
- 음식이 쌓이면(정체/가스/염증 등) **구간별 트러블**이 발생
- 소화에 실패한 음식은 **장에 막힌 상태로 잔류**해 길을 좁히거나 막음(자동으로 경로에서 사라지지 않음)
- 기관(구간)마다 규칙이 달라서 **타워 조합/배치가 핵심 퍼즐**

---

## 핵심 게임 루프

1. 소화기관 2D 그리드 맵 생성(밥 위/디저트 위/술 위/소장/대장 구간 포함)
2. 음식(Enemy)이 입구에서 생성되어 기관 경로를 따라 이동
3. 빌드 타일에 소화 보조 타워(효소/산/담즙/프로바이오틱스 등) 배치
4. 타워가 음식에 "소화 피해" 또는 상태이상(분해/중화/유화/흡수 촉진)을 부여
5. 음식이 쌓이거나 특정 음식 조합이 누적되면 구간 트러블(역류/과산/염증/변비 등) 발생
6. 라운드 종료 후 영양(자원) 획득 → 타워 강화/신규 타워 해금/기관 업그레이드
7. 재접속 시 "식사 체크인"으로 오늘 먹은 음식을 선택 → 해당 음식 웨이브로 한 번에 몰아치는 도전(Idle 보상)

## 맵 구조 (소화기관)

- **밥 위(Rice Stomach)**: 기본 유입량이 많아 범용 소화 안정성이 중요한 구간
- **디저트 위(Dessert Stomach)**: 당류/유제품 계열 밀도가 높아 순간 병목이 자주 발생하는 구간
- **술 위(Alcohol Stomach)**: 자극성/산도 변동이 커 트러블 관리가 핵심인 구간
- **소장(Small Intestine)**: 분기/곡선이 많고 길며, 흡수/지원 타워의 가치가 큼
- **대장(Large Intestine)**: 수분/섬유질/발효 중심, 정체가 길어지면 "변비" 트러블 발생

타일 예시:

- Path Tile: 음식 이동 경로
- Buildable Tile: 타워 설치 가능
- Organ Tile: 구간 고유 규칙/버프/디버프 적용
- Special Tile: 체크포인트(괄약근), 보스 이벤트, 상점 등

## 음식(Enemy) 시스템

음식은 단순한 HP 덩어리가 아니라 "소화 난이도"를 가진 개체입니다.

- **탄수화물**: 분해는 쉽지만 대량 유입 시 급격히 쌓임
- **지방식**: 이동/분해가 느리고 유화(담즙) 없으면 강함
- **단백질**: 위 구간(산/효소)에서 잘 처리되며, 처리 실패 시 잔여물이 트러블 유발
- **유제품/매운 음식/탄산**: 특정 트러블(염증/가스/과산 등)을 빠르게 누적
- **보스 음식**: "특대 피자", "치즈버거 세트" 같은 복합 속성 + 저항

핵심 메커니즘:

- 음식이 **쌓이면(Stack)** 구간 압력이 올라가고, 트러블 발생 확률이 증가
- 소화하지 못한 음식은 **그 자리에서 잔류(막힘)**하며, 플레이어가 처리할 때까지 자동으로 빠져나가지 않음
- 구간별로 "잘 맞는 처리법"이 달라서, 타워 선택이 중요

## 소화 보조 타워(디펜스) 컨셉

타워는 "무기"가 아니라 **소화 과정의 역할**을 수행합니다.

- **효소 분사기(Enzyme)**: 특정 음식 타입(탄수/단백/지방) 분해 보너스
- **위산 분무(Acid)**: 위 구간에서 강력, 대신 과산 누적 리스크
- **담즙 제트(Bile)**: 지방 유화/약화, 범위 디버프
- **프로바이오틱스 구름(Probiotic)**: 대장 구간 버프, 발효/부패 트러블 완화
- **연동 운동 펄스(Peristalsis)**: 음식 밀어내기/정체 해소(지원형)
- **흡수 필터(Absorption)**: 소장에서 자원(영양) 획득량 증가(딜+경제 하이브리드)

타겟팅(예):

- First / Last
- 가장 많이 쌓인 군집(Cluster)
- 특정 속성 우선(지방/유제품 등)

## 자원/성장(Idle 포함)

- 전투 보상: 영양(Nutrients), 에너지(Energy), 장내균 배양치(Microbiome) 등
- 기관 업그레이드: 구간별 기본 능력(산도 안정, 흡수 효율, 수분 조절 등) 강화
- 오프라인 보상(방치): "식사 체크인"으로 **한꺼번에 많은 적을 처치할 기회**를 제공

### 방치(오프라인) 설계: 식사 체크인

이 게임의 방치 보상은 자원을 자동으로 쌓아두는 방식이 아니라,
**유저가 현실에서 먹은 음식 기록을 게임의 웨이브로 바꿔서 한 번에 몰아치는 도전 기회**를 주는 방식입니다.

재접속 흐름:

1. 게임이 마지막 접속 시간을 기준으로 경과 시간을 체크
2. 하루 식사 슬롯이 열려 있으면(시간이 경과했으면) 팝업을 표시
3. 팝업 질문: **"오늘 뭐 드셨어요?"**
4. 유저는 식사 슬롯별로 메뉴를 선택(아침 → 점심 → 오후 간식 → 저녁 → 야식)
5. 체크인을 마치고 게임을 재개하면:
   - 기존처럼 랜덤 음식이 단일 시작점에서 내려오는 것이 아니라
   - **유저가 선택한 메뉴들이 집중적으로 등장**하는 "오늘의 웨이브"가 구성됨
6. 한 번의 웨이브(Enemy 묶음)를 소화(클리어)하는 것이 오늘의 도전/보상 단위가 됨

핵심 의도:

- "오프라인 보상"을 숫자 정산이 아니라, **플레이할 이유가 되는 전투 콘텐츠**로 변환
- 유저의 현실 행동(먹은 메뉴)이 곧 게임 난이도/상성/빌드 고민으로 이어지게 구성
- 하루 루틴(식사 5회)과 맞물려 자연스럽게 재접속 동기를 제공

## 기술(프로젝트) 방향

- **WebGL2 기반**: 고성능 2D 렌더링
- **모듈화 구조**: ES6 Modules로 깔끔한 코드 분리
- **Local-first**: 서버 없이 실행 가능
- **저장**: localStorage / IndexedDB 스냅샷(맵/음식/타워/자원/타임스탬프)
- **최적화**: 캐싱, 배칭, 인스턴싱 등 성능 최적화 적용

## 개발 가이드

### 새 모듈 추가하기

1. 적절한 폴더에 모듈 생성 (renderer/, game/, systems/, utils/)
2. ES6 export/import 사용
3. config.js에 설정 상수 추가
4. main.js에서 초기화

### BGM 생성하기

게임 테마에 맞는 새로운 BGM을 생성하려면:

```bash
# 기본 생성 (매번 다른 변주)
python src/utils/chiptune_generator.py --bpm 145 --seed 2026

# 다른 템포/시드로 생성
python src/utils/chiptune_generator.py --bpm 160 --seed 42
```

생성된 파일:
- `src/assets/bgm/game_theme.wav` - 메인 BGM (자동 루프)
- `src/assets/bgm/game_theme_structure.json` - 곡 구조 메타데이터
- `src/assets/bgm/instruments/*.wav` - 악기별 WAV 파일 (18개)

**곡 구조 (음식의 즐거움 테마)**

C major 키로 작곡된 밝고 경쾌한 16-bit 칩튠:

1. **INTRO** - 음식 앞의 설렘 (drums + bass, 통통 튀는 리듬)
2. **BUILD** - 첫 입 맛보기 (+ pad, arpeggio, 기대감 상승)
3. **VERSE** - 냠냠냠 맛있다! (+ melody, harmony, 경쾌한 멜로디)
4. **CHORUS** - 맛의 정점! (전체 악기 합주, 100% 에너지)
5. **BRIDGE** - 여운과 소화 (pad + arpeggio + bass, 차분한 휴식)
6. **CHORUS2** - 또 먹고 싶다! (다시 전체 합주, 변주 C)
7. **OUTRO** - 만족스러운 마무리 (melody + pad + arpeggio)

**음악적 특징:**
- 밝고 긍정적인 C major 키
- 경쾌한 스퀘어파 멜로디 (먹는 즐거움)
- 통통 튀는 베이스 라인 (소화의 리듬)
- 반짝이는 아르페지오 (음식의 반짝임)

### 성능 최적화 팁

- WebGL: Static mesh 사용, draw call 최소화
- Canvas 2D: 캐싱 활용, dirty rect 최적화
- 메모리: TypedArray 재사용, 객체 풀링
- Audio: WAV 압축, Web Audio API 활용
